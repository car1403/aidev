# 01 Topic Selection

팀 프로젝트의 첫 단계는 구현 가능한 주제를 정하는 것입니다.

이번 과정의 기본 주제는 **실시간 로그 대시보드 인터페이스**입니다. 이전에 배운 기능을 무리하게 모두 넣기보다, 같은 구조 안에서 어떤 로그를 수집하고 어떤 화면으로 보여줄지 정하는 것을 권장합니다.

## 기본 주제

```text
웹 서비스 기초 및 AI 백엔드 개발
-> 실시간 로그 대시보드 인터페이스
```

## 주제 선정 원칙

좋은 주제는 기능이 많은 주제가 아니라 데이터 흐름이 분명한 주제입니다.

```text
사용자 입력
-> FastAPI API
-> Supabase 저장
-> Streamlit 대시보드 조회
-> 피드백 또는 상태 변경
```

## 좋은 주제의 조건

- 대상 사용자가 명확합니다.
- 어떤 로그를 수집할지 설명할 수 있습니다.
- 로그가 왜 필요한지 설명할 수 있습니다.
- 핵심 기능은 2~4개 정도입니다.
- FastAPI와 Streamlit을 연결할 이유가 있습니다.
- Supabase에 저장할 테이블이 있습니다.
- 시연 가능한 대시보드 화면이 있습니다.
- SSE는 필수 구현이 아니라 선택 확장 또는 설계 문서 반영으로도 가능합니다.

## 추천 변형 주제

- AI 질문/응답 로그 대시보드
- 학습 기록과 AI 피드백 로그 대시보드
- 고객 문의 처리 로그 대시보드
- 오류 발생 및 해결 상태 추적 대시보드
- 사용자 피드백 기반 AI 응답 품질 대시보드
- 업무 요청 처리 상태 로그 대시보드

## 주제 선정 양식

```text
프로젝트 이름:
대상 사용자:
수집할 로그:
로그를 수집하는 이유:
핵심 기능 1:
핵심 기능 2:
핵심 기능 3:
Supabase에 저장할 데이터:
대시보드에서 보여줄 지표:
AI 기능 사용 여부:
SSE 적용 여부:
시연 시나리오:
```

## 범위 조정 기준

처음부터 모든 기능을 구현하려고 하지 않습니다. 먼저 다음 질문에 답합니다.

- 첫날 안에 첫 화면과 첫 API를 만들 수 있는가?
- 둘째 날 안에 Supabase 저장과 조회가 가능한가?
- 셋째 날 오전에 대시보드 시연을 준비할 수 있는가?
- SSE가 어렵다면 `st.spinner`, 새로고침, 상태 표시 방식으로 대체할 수 있는가?

이 질문에 답하기 어렵다면 주제를 줄이는 것이 좋습니다.
