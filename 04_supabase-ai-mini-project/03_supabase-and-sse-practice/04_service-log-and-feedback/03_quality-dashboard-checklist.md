# 03. 품질 대시보드 체크리스트

품질 대시보드는 프로젝트 결과를 설명할 때 사용할 수 있는 운영 화면입니다.

## 포함하면 좋은 항목

```text
전체 질문 수
전체 AI 응답 수
최근 오류 수
평균 피드백 점수
최근 서비스 로그
최근 사용자 피드백
```

## 결과 문서에 정리할 내용

```text
어떤 로그를 수집했는가?
오류가 발생하면 어떻게 표시되는가?
피드백을 어떻게 저장했는가?
피드백을 바탕으로 무엇을 개선할 수 있는가?
```

## 발표 시 확인할 질문

- 대시보드가 실제 실행 데이터와 연결되어 있습니까?
- 화면에 표시된 수치가 어떤 테이블에서 온 것인지 설명할 수 있습니까?
- 오류와 성공 로그가 구분되어 있습니까?
- 피드백이 프로젝트 개선 방향과 연결되어 있습니까?
