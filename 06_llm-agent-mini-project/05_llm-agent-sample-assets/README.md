# 65 LLM Agent Sample Assets

이 폴더는 65 미니 프로젝트를 진행할 때 참고할 수 있는 샘플 자산을 정리하는 위치입니다.

실제 실행 가능한 기본 샘플은 `62_instructor-sample-project`에 있고, 팀별 프로젝트 시작 템플릿은 `99_team-projects/llm-agent-team-template`에 있습니다. 이 폴더는 그 사이에서 설계 아이디어, 요청 예시, Tool 설계 예시, 발표 참고 자료를 모아두는 보조 자료 역할을 합니다.

## 이 폴더에서 다루면 좋은 자료

| 자료 | 쓰임 |
| --- | --- |
| 샘플 사용자 요청 목록 | 에이전트가 처리해야 할 입력 예시를 미리 정리합니다. |
| Mock data 예시 | 외부 API 없이도 Tool 호출 흐름을 테스트할 수 있게 합니다. |
| Tool schema 예시 | Tool의 입력값, 출력값, 실패 상황을 설계할 때 참고합니다. |
| LangGraph 흐름도 | Start, Decision, Tools, Review, End 노드의 연결을 시각적으로 정리합니다. |
| 테스트 시나리오 예시 | 정상 요청, 정보 부족, Tool 실패, 응답 불일치 상황을 검증합니다. |
| 발표 자료 예시 | 최종 발표에서 어떤 순서로 설명하면 좋은지 참고합니다. |

## 65 과정에서 중요한 기준

65 과정은 64에서 배운 Prompt, Tool Use, Memory, LangGraph, Self-Reflection 개념을 하나의 작은 프로젝트에 적용하는 단계입니다.

따라서 샘플 자산을 만들 때도 단순히 보기 좋은 자료를 모으는 것보다 다음 질문에 답할 수 있어야 합니다.

- 에이전트가 어떤 문제를 해결하는가?
- 어떤 정보를 State에 저장하는가?
- 어떤 기준으로 Tool을 선택하는가?
- Tool 실패나 응답 불일치가 생기면 어떻게 복구하는가?
- 최종 결과가 이전보다 좋아졌다는 것을 어떻게 확인하는가?

## 확장 기준

처음에는 Mock data와 로컬 실행만으로 충분합니다. OpenAI API, Ollama 기반 Llama, pgvector 기반 장기 기억은 프로젝트가 안정적으로 동작한 뒤 선택적으로 확장합니다.

Docker Compose, AWS 배포, GitHub Actions, 운영 자동화는 07 과정에서 본격적으로 다룹니다.
