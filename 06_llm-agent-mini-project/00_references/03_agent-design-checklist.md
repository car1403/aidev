# Agent Design Checklist

프로젝트를 시작하기 전에 아래 항목을 먼저 작성합니다.

## 사용자 요청

- 사용자는 어떤 문장을 입력하는가?
- 성공한 응답 예시는 무엇인가?
- 실패하거나 애매한 요청은 무엇인가?

## State

- `user_request`: 원본 요청
- `intent`: 분석된 의도
- `required_tools`: 필요한 도구 목록
- `tools_called`: 실제 호출한 도구 목록
- `tool_results`: 도구 실행 결과
- `error_count`: 오류 또는 재시도 횟수
- `iteration`: 반복 실행 횟수
- `memory_summary`: 검색 또는 메모리 요약 결과
- `final_answer`: 사용자에게 보여줄 최종 응답

## Tool

- Tool 이름은 명확한가?
- 입력값과 출력값이 정해져 있는가?
- 실패했을 때 반환값이 있는가?
- Tool이 외부 API 없이 Mock data로 먼저 동작하는가?
- Tool 실행 결과를 사람이 읽을 수 있는 형태로 로그에 남기는가?

## Graph

- 시작 노드는 무엇인가?
- 요청을 인지하는 노드는 무엇인가?
- 도구 사용 여부를 판단하는 노드는 무엇인가?
- Tool 실행 노드는 어디인가?
- 결과를 검증하는 노드는 어디인가?
- 재시도 또는 fallback 노드는 어디인가?
- 응답 생성 노드는 어디인가?
- 조건 분기가 필요한가?

Graph는 가능하면 아래 사고 흐름으로 설명합니다.

```text
인지 -> 판단 -> 행동 -> 검증 -> 최종 응답
```

## UI

- 사용자는 어디에 요청을 입력하는가?
- 최종 답변은 어디에 표시하는가?
- Agent State를 디버깅용으로 보여줄 것인가?

## 테스트와 발표

- 정상 요청 2개 이상을 실행했는가?
- 애매한 요청 또는 실패 요청 1개 이상을 실행했는가?
- Tool 선택이 틀렸을 때 다시 시도하거나 fallback하는가?
- 최종 발표에서 State, Tool, Graph 흐름을 한 장으로 설명할 수 있는가?
