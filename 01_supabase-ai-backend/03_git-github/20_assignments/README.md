# 20. Assignments

이 폴더는 Git/GitHub 단원의 과제 안내를 담고 있습니다.

과제의 목표는 Git 명령을 외우는 것이 아니라, 변경 내용을 확인하고 안전하게 커밋할 수 있는 습관을 만드는 것입니다.

## Assignment 01. Git Status Report

제출 내용:

- `git status` 결과를 보고 변경된 파일을 설명합니다.
- 추적되는 파일과 추적되지 않는 파일의 차이를 설명합니다.
- `.venv`, `.env`가 Git에 올라가면 안 되는 이유를 작성합니다.

실제 key 값이나 token 값은 제출하지 않습니다.

## Assignment 02. Commit Message

제출 내용:

- 본인이 수정한 내용 요약
- 커밋 메시지 후보 3개
- 가장 적절하다고 생각한 커밋 메시지 1개와 이유

좋은 커밋 메시지 예시:

```text
docs: add beginner Git workflow guide
fix: update Supabase setup path
feat: add Upstash Redis practice notes
```

## Assignment 03. README Update

제출 내용:

- 본인이 작성한 README 초안
- README에 포함한 핵심 내용
- 수정 전후 차이
- README를 읽는 사람이 바로 실행할 수 있는지에 대한 점검 결과

## Assignment 04. Secret Safety Check

제출 내용:

- GitHub에 올리면 안 되는 파일 목록
- `.env.example`과 `.env`의 차이
- API key가 노출되었을 때 해야 할 조치
- 본인의 과제 폴더에 민감정보가 없는지 점검한 결과

## Assignment 99. Git Learning Log

제출 내용:

```text
1. 이번 단원에서 배운 Git 명령 5개
2. 가장 헷갈렸던 개념 1개
3. VS Code Source Control에서 확인한 기능 3개
4. GitHub에 올리면 안 되는 파일 예시
5. 본인이 생각하는 좋은 커밋 습관
```

제출물은 Markdown 문서로 작성합니다.
