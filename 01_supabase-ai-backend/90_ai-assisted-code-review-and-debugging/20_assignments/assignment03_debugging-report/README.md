# Assignment 03 - 디버깅 보고서

이 과제에서는 오류가 있는 코드를 Codex와 함께 분석하고, 수정 과정을 보고서로 정리합니다.

디버깅 보고서의 핵심은 "오류를 고쳤다"가 아니라 "어떤 오류였고, 왜 발생했으며, 어떻게 확인했는가"를 설명하는 것입니다.

## 목표

- 실행 명령과 오류 메시지를 정확히 기록할 수 있다.
- Codex에게 오류 분석을 요청할 수 있다.
- 수정 전 코드와 수정 후 코드를 비교할 수 있다.
- 오류 원인을 본인 말로 설명할 수 있다.
- 수정 후 같은 명령으로 다시 확인할 수 있다.

## 수행 절차

1. 오류가 있는 Python 코드를 준비합니다.
2. 실행 명령을 기록합니다.
3. 오류 메시지를 생략하지 않고 복사합니다.
4. Codex에게 원인 후보와 확인 순서를 요청합니다.
5. 수정 전 코드를 `before.py`로 저장합니다.
6. 수정 후 코드를 `after.py`로 저장합니다.
7. 수정 후 같은 명령으로 다시 실행합니다.
8. 오류 원인과 해결 과정을 보고서에 작성합니다.

## 제출 파일

```text
debugging-report.md
before.py
after.py
```

## Codex 요청 예시

```text
Python 코드를 실행했더니 오류가 발생했습니다.

실행 명령:
python before.py

오류 메시지:
여기에 오류 메시지를 붙여 넣습니다.

요청:
1. 오류 메시지를 초보자 기준으로 설명해주세요.
2. 오류가 발생한 줄과 원인 후보를 알려주세요.
3. 바로 코드를 고치기 전에 확인할 순서를 정리해주세요.
4. 수정 예시를 보여주세요.
5. 수정 후 어떻게 다시 확인하면 되는지 알려주세요.
```
