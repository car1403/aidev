# AI Tool Selection Guide

AI 보조 개발에서는 하나의 도구만 사용하는 것보다, 목적에 맞는 도구를 선택하는 능력이 중요합니다.

이 문서는 Codex, Claude Code, NotebookLM, Perplexity AI, 일반 챗봇형 모델을 어떤 상황에서 사용할지 정리합니다.

## 도구별 역할

| 도구 | 잘 맞는 작업 | 수업에서의 예 |
| --- | --- | --- |
| Codex | 로컬 파일을 읽고 수정하며 코드 실행 결과를 바탕으로 개선 | FastAPI 예제 파일에 주석을 추가하고 테스트를 실행합니다. |
| Claude Code | 코드 설명, 긴 맥락 기반 리뷰, 대안 설계 비교 | 프로젝트 구조나 리팩토링 방향을 비교할 때 참고합니다. |
| NotebookLM | 업로드한 문서 기반 요약과 질문 생성 | 강의 README를 넣고 복습 질문을 만듭니다. |
| Perplexity AI | 웹 검색 기반 답변과 출처 확인 | FastAPI, Supabase, Upstash Redis 공식 문서를 찾습니다. |
| ChatGPT/Claude/Gemini | 개념 설명, 아이디어 정리, 코드 초안 생성 | Python 문법, API 설계, 오류 메시지 설명을 요청합니다. |

## 개발 단계별 추천 흐름

```text
1. 개념 이해
   -> ChatGPT/Claude/Gemini에게 쉬운 설명을 요청합니다.

2. 현재 프로젝트 파일 수정
   -> Codex에게 파일 위치와 원하는 변경 사항을 알려줍니다.

3. 긴 문서 복습
   -> NotebookLM으로 강의 문서 기반 질문을 만듭니다.

4. 최신 정보 확인
   -> Perplexity AI 또는 공식 문서로 출처를 확인합니다.

5. 코드 리뷰
   -> Codex 또는 Claude Code에게 실행 안정성, 보안, 가독성 관점의 리뷰를 요청합니다.
```

## 답변 검증 기준

AI 도구가 만든 답변은 다음 기준으로 확인합니다.

- 현재 폴더 구조와 파일명이 맞는가?
- 실제 터미널에서 실행되는가?
- `.env`나 API key 같은 민감 정보가 노출되지 않았는가?
- 공식 문서와 충돌하는 내용은 없는가?
- 비용이 발생할 수 있는 API 호출이 통제되어 있는가?
- 초보자가 읽을 수 있는 주석과 설명이 포함되어 있는가?

## 수업용 질문 예시

```text
Codex에게:
이 파일을 초보자가 읽기 쉽게 주석을 추가해 주세요.
수정 후 실행 가능한지 확인해 주세요.

Perplexity AI에게:
FastAPI Depends 공식 문서 위치를 찾아주고 핵심 개념을 요약해 주세요.

NotebookLM에게:
이 단원의 README를 기반으로 학습 퀴즈 10개를 만들어 주세요.
```
