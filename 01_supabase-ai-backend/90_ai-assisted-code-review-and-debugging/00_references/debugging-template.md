# 디버깅 요청 템플릿

오류가 발생하면 코드만 붙여 넣기보다 실행 명령, 오류 메시지, 기대 결과를 함께 전달해야 합니다.

## 기본 디버깅 요청 템플릿

```text
문제 상황:

실행한 위치:

실행한 명령:

오류 메시지:

관련 코드:

기대한 결과:

실제 결과:

내가 이미 시도한 것:

요청:
1. 오류 메시지를 초보자 기준으로 설명해주세요.
2. 원인 후보를 단계별로 나누어 주세요.
3. 가장 먼저 확인할 것을 알려주세요.
4. 수정 방법과 재확인 방법을 알려주세요.
```

## 백엔드 디버깅 요청 템플릿

```text
문제 상황:
FastAPI/Supabase/Upstash Redis 실습 중 오류가 발생했습니다.

실행 위치:

실행한 명령:

오류 메시지:

관련 파일:

.env에 설정한 변수 이름:
실제 key 값은 붙이지 않습니다.

기대한 결과:

실제 결과:

확인한 것:
- 가상환경 활성화 여부
- requirements.txt 설치 여부
- Supabase 테이블 생성 여부
- Supabase 테이블명과 컬럼명
- Upstash Redis URL/token 설정 여부
- Redis TTL 설정 여부

요청:
1. 오류 원인을 단계별로 추정해주세요.
2. 코드 문제와 설정 문제를 나누어 설명해주세요.
3. 초보자가 따라 할 수 있는 확인 순서를 알려주세요.
4. 실제 key를 노출하지 않고 확인하는 방법을 알려주세요.
5. 수정 후 재실행 방법을 알려주세요.
```
