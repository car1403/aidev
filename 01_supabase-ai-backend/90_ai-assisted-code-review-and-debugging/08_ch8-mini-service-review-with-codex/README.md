# 08_ch8-mini-service-review-with-codex

`08_backend-mini-service-practice`에서 만든 미니 서비스를 Codex와 함께 리뷰하는 단원입니다.

## 학습 목표

- 요구사항과 구현 결과가 일치하는지 확인합니다.
- mock 구현과 Supabase 구현의 차이를 설명합니다.
- endpoint 응답 형식이 일관적인지 리뷰합니다.
- 서비스 로그가 충분한 정보를 담는지 점검합니다.
- 최종 프로젝트 전에 개선할 부분을 정리합니다.

## 리뷰 대상 파일

```text
08_backend-mini-service-practice/04_implementation-guide/main_mock.py
08_backend-mini-service-practice/04_implementation-guide/main_supabase.py
08_backend-mini-service-practice/04_implementation-guide/schemas.py
08_backend-mini-service-practice/03_supabase-schema/mini-service-schema.sql
```

## Codex 요청 예시

```text
08_backend-mini-service-practice의 미니 서비스를 리뷰해줘.

리뷰 관점:
1. 요구사항과 endpoint가 일치하는가?
2. Pydantic 모델의 검증 조건이 충분한가?
3. mock 저장소와 Supabase 저장소의 차이가 명확한가?
4. 서비스 로그가 운영에 필요한 정보를 담고 있는가?
5. 최종 프로젝트로 확장하려면 무엇을 개선해야 하는가?
```
