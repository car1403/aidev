# Assignment 01 - 서비스 데이터 저장 위치 분류

서비스에서 다루는 데이터를 Supabase에 저장할지, Upstash Redis에 저장할지 분류하는 과제입니다.

## 목표

- 영구 저장 데이터와 임시 데이터를 구분할 수 있습니다.
- Supabase와 Upstash Redis의 역할을 설명할 수 있습니다.
- 저장 위치를 선택한 이유를 서비스 관점에서 설명할 수 있습니다.

## 분류할 데이터

아래 데이터를 각각 Supabase 또는 Upstash Redis로 분류합니다.

```text
사용자 프로필
대화 이력
AI 답변 피드백
최근 1분 요청 횟수
30초 검색 결과 캐시
서비스 오류 로그
로그인 중 임시 상태
중복 요청 방지 key
```

## 제출물

아래 표 형태로 작성합니다.

| 데이터 | 저장 위치 | 이유 |
|---|---|---|
| 사용자 프로필 | Supabase | 오래 보관하고 다시 조회해야 하기 때문 |
| 최근 1분 요청 횟수 | Upstash Redis | 시간이 지나면 자동 만료되어도 되기 때문 |

## 확인 기준

- 오래 보관해야 하는 데이터는 Supabase로 분류했습니다.
- TTL이 필요한 데이터는 Upstash Redis로 분류했습니다.
- 각 항목마다 이유를 한 문장 이상 작성했습니다.

- 서비스 오류 로그를 Supabase에 저장하면 어떤 장점이 있나요?
