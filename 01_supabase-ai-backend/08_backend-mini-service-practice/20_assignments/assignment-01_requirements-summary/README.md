`AI 질문 응답 백엔드 미니 서비스`가 무엇을 만들고자 하는지 요구사항을 정리합니다.

## 제출 내용

아래 항목을 문서로 정리합니다.

| 항목 | 작성 내용 |
| --- | --- |
| 서비스 이름 | 미니 서비스 이름 |
| 서비스 목적 | 이 서비스가 해결하는 문제 |
| 사용자 시나리오 | 사용자가 질문을 보내고 답변을 받는 흐름 |
| 필수 기능 | 상태 확인, 질문 등록, 기록 조회, 로그 저장, 오류 응답 등 |
| 선택 기능 | Gemini API 연동, OpenAI 비교, Redis 캐시, 요청 제한 등 |
| 제외 범위 | SSE, 프론트엔드, Docker/AWS 배포 등 이번 단계에서 깊게 다루지 않는 내용 |

## 작성 예시

```text
서비스 목적:
사용자가 질문을 보내면 FastAPI 백엔드가 질문을 검증하고 mock AI 답변을 생성한 뒤,
질문/답변 기록과 서비스 로그를 Supabase에 저장한다.
```

## 완료 기준

- 서비스 목적이 한 문장으로 정리되어 있습니다.
- 필수 기능과 선택 기능이 구분되어 있습니다.
- Supabase와 Upstash Redis의 역할이 구분되어 있습니다.
- 제외 범위가 명확하게 적혀 있습니다.
