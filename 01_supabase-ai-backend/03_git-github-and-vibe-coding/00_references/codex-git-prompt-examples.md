# Codex Git Prompt Examples

Codex는 Git 명령을 무작정 대신 실행하는 도구가 아니라, 변경 내용을 이해하고 설명을 돕는 학습 파트너로 사용합니다.

## diff 설명 요청

```text
아래 git diff를 초보자에게 설명하듯이 정리해 주세요.
무엇이 추가되었고, 왜 필요한지 항목별로 설명해 주세요.
```

## 커밋 메시지 요청

```text
현재 변경 내용을 기준으로 커밋 메시지 후보를 5개 작성해 주세요.
docs, feat, fix 중 어떤 type이 적절한지도 설명해 주세요.
```

## README 개선 요청

```text
이 README가 학생들이 수업 시간에 따라 보기 충분한지 검토해 주세요.
부족한 설명, 실행 순서, 주의사항을 보강해 주세요.
```

## 민감정보 점검 요청

```text
아래 변경 내용에 API key, token, password 같은 민감정보가 포함되어 있는지 점검해 주세요.
.env와 .env.example 사용 기준도 함께 확인해 주세요.
```

## 오류 메시지 설명 요청

```text
아래 Git 오류 메시지를 초보자가 이해할 수 있게 설명해 주세요.
원인, 확인 명령, 해결 순서로 나누어 주세요.
```

## 좋은 질문 형식

Codex에게 질문할 때는 아래 정보를 함께 주면 좋습니다.

```text
1. 무엇을 하려고 했는지
2. 어떤 명령을 실행했는지
3. 어떤 오류가 나왔는지
4. 현재 폴더 위치가 어디인지
5. 원하는 결과가 무엇인지
```
