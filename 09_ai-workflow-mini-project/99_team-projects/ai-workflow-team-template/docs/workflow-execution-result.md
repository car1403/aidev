## 2. 정상 시나리오 테스트

| 항목 | 내용 |
| --- | --- |
| 테스트 이름 | VPN 접속 오류 문의 처리 |
| 입력 데이터 | VPN 접속이 되지 않습니다. 오류 코드 809가 표시됩니다. |
| 기대 결과 | 네트워크 장애 유형으로 분류하고 관련 매뉴얼을 검색해 답변 초안을 생성한다. |
| 실제 결과 | |
| 성공 여부 | 성공 / 실패 |

실행 흐름:

```text
Trigger Node
-> Classifier Node
-> RAG Search Node
-> Response Node
-> Notify Node
-> Log Node
```

실행 로그:

```text
[09:00:01] ticket received: TCK-001
[09:00:02] category classified: network
[09:00:03] retrieved docs: vpn-guide-01, vpn-error-03
[09:00:05] answer generated
[09:00:06] slack notification sent
```

## 3. 예외 시나리오 테스트

| 테스트 | 입력 또는 상황 | 기대 결과 | 실제 결과 | 성공 여부 |
| --- | --- | --- | --- | --- |
| 필수 입력 누락 | message 없음 | 다시 입력 요청 | | |
| 인증 실패 | 잘못된 API Key | 관리자 확인 안내 | | |
| 검색 결과 없음 | 관련 문서 없음 | fallback 답변 생성 | | |
| 외부 API 실패 | Slack 5xx 응답 | 재시도 후 Email 대체 | | |
| 부적절한 입력 | Prompt Injection 시도 | 입력 차단 또는 안전 답변 | | |

각 예외 테스트에는 가능하면 아래 내용을 함께 기록합니다.

