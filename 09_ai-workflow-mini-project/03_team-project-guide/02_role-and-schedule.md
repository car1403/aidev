# 02. Role And Schedule

팀 프로젝트는 역할을 나누어 진행하면 훨씬 안정적으로 완성할 수 있습니다.

## 1. 권장 역할

| 역할 | 담당 작업 |
| --- | --- |
| PM / 발표 담당 | 주제 범위 조정, 일정 관리, 최종 발표 자료 정리 |
| 워크플로우 설계 담당 | Trigger, 처리 노드, 출력, 분기, 반복, 예외 흐름 설계 |
| RAG 설계 담당 | 문서 수집, chunking 전략, embedding, vector DB 구조 정리 |
| 도구 연동 담당 | AIPP, n8n, Dify, Slack, Email, Jira, Notion, API 연동 설계 |
| 테스트 담당 | 정상/예외 시나리오 테스트, 실행 로그 정리, 결과 보고서 작성 |

팀원이 적다면 한 사람이 여러 역할을 맡아도 됩니다. 중요한 것은 각 산출물의 책임자가 분명해야 한다는 점입니다.

## 3. 팀 회의 체크리스트

매 수업 또는 작업 시간마다 아래 질문을 확인합니다.

- 오늘 작성해야 할 문서는 무엇입니까?
- 현재 워크플로우에서 입력과 출력이 명확합니까?
- RAG가 필요한 지점과 필요하지 않은 지점이 구분되어 있습니까?
- 외부 서비스 연동에 인증과 오류 처리가 포함되어 있습니까?
- 테스트할 정상 케이스와 예외 케이스가 정리되어 있습니까?
