# 04. Project Checklist

미니 프로젝트와 팀 프로젝트 공통 체크리스트입니다.

## 기획

```text
[ ] 프로젝트 주제가 명확하다.
[ ] 사용자가 누구인지 정의했다.
[ ] 해결하려는 문제가 구체적이다.
[ ] 입력과 출력이 정리되어 있다.
```

## 구현

```text
[ ] README.md가 있다.
[ ] 실행 방법이 작성되어 있다.
[ ] requirements.txt가 있다.
[ ].env.example이 있다.
[ ] 실제.env는 제출하지 않는다.
[ ] 주요 API가 동작한다.
[ ] 화면에서 API 호출 결과가 보인다.
```

## Supabase 프로젝트

```text
[ ] 테이블 설계 문서가 있다.
[ ] API key 사용 위치가 구분되어 있다.
[ ] service role key가 프론트엔드에 없다.
[ ] RLS 사용 여부와 이유가 작성되어 있다.
```

## AI/Streaming

```text
[ ] AI 응답 기능이 있다.
[ ] 일반 응답 API 또는 스트리밍 응답 API가 있다.
[ ] SSE를 적용한 경우 최종 응답 저장 기준이 있다.
[ ] 오류 발생 시 fallback 메시지가 있다.
```

## 운영/보안

```text
[ ] 로그 항목이 정의되어 있다.
[ ] 비용/API 사용량 관리 기준이 있다.
[ ] 민감정보를 더미 데이터로 대체했다.
[ ] API Key가 코드나 문서에 직접 들어 있지 않다.
```
