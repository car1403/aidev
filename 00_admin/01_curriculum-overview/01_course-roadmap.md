# 01. Course Roadmap

이 문서는 01~09 과정 전체 로드맵입니다.

## 전체 로드맵

```text
01_supabase-ai-backend
-> 02_supabase-ai-frontend
-> 03_supabase-ai-mini-project
-> 04_llm-agent-orchestration
-> 05_llm-agent-mini-project
-> 06_multi-agent-service-ops
-> 07_multi-agent-service-mini-project
-> 08_ai-workflow-automation
-> 09_ai-workflow-mini-project
```

## 단계별 의미

| 단계 | 과정 | 의미 |
| --- | --- | --- |
| 01 | Supabase AI Backend | Python과 FastAPI로 서버를 만들고 Supabase에 데이터를 저장합니다. |
| 02 | Supabase AI Frontend | Streamlit으로 화면을 만들고 백엔드 API를 호출합니다. |
| 03 | Supabase AI Mini Project | 백엔드, 프론트엔드, Supabase를 연결해 작은 AI 서비스를 만듭니다. |
| 04 | LLM Agent Orchestration | Docker 기반으로 LLM, Tool, Memory, RAG, Agent 흐름을 배웁니다. |
| 05 | LLM Agent Mini Project | 04에서 배운 Agent 구조로 미니 프로젝트를 만듭니다. |
| 06 | Multi-Agent Service Ops | Docker Compose, AWS, GitHub Actions, 운영/보안/모니터링을 배웁니다. |
| 07 | Multi-Agent Service Mini Project | 운영 관점의 Multi-Agent 미니 프로젝트를 만듭니다. |
| 08 | AI Workflow Automation | AIPP, n8n, Dify 기반 AI Workflow를 설계합니다. |
| 09 | AI Workflow Mini Project | AI Workflow 기반 최종 미니 프로젝트를 만듭니다. |

## 학습 단계

```text
기초 문법
-> API 서버
-> 화면 UI
-> Supabase 저장
-> AI 응답
-> 실시간 스트리밍
-> Agent 구조
-> 운영 자동화
-> Workflow 자동화
-> 팀 프로젝트
```

## 학생에게 설명할 핵심

처음부터 모든 기술을 한 번에 이해할 필요는 없습니다.

```text
01~03에서는 "작은 AI 서비스 하나를 직접 만든다"가 목표입니다.
04~05에서는 "AI가 Tool과 Memory를 활용하는 구조"를 배웁니다.
06~07에서는 "서비스를 운영 가능한 형태로 만드는 방법"을 배웁니다.
08~09에서는 "노코드/로코드 도구로 업무 자동화 Workflow를 설계"합니다.
```
