# 05. Time Management Guide

하루 8시간 수업을 기준으로 시간을 관리하기 위한 운영 안내입니다.

이 문서는 모든 과정을 같은 흐름으로 진행하기 위한 기준입니다. 실제 수업에서는 과정 난이도, 설치 상황, 프로젝트 진행 속도에 따라 일부 시간을 조정할 수 있습니다.

## 하루 8시간 기본 구조

아래 예시는 점심시간을 제외한 실제 수업 8시간 기준입니다.

```text
09:00~09:20: 오늘 목표와 폴더 위치 확인
09:20~10:00: 핵심 개념 설명
10:00~10:50: 첫 번째 시연
10:50~11:00: 휴식
11:00~12:00: 기본 실습
12:00~13:00: 점심
13:00~13:20: 오전 실습 상태 점검
13:20~14:20: 두 번째 시연 또는 확장 개념 설명
14:20~15:20: 확장 실습
15:20~15:30: 휴식
15:30~16:30: Lab 또는 Assignment 진행
16:30~17:20: 오류 해결, 코드 리뷰, 문서 정리
17:20~17:50: 체크리스트 확인과 제출물 정리
17:50~18:00: 다음 수업 연결 안내
```

## 일반 실습일 운영 기준

새로운 개념과 예제를 배우는 날은 아래 비율을 기준으로 진행합니다.

```text
개념 설명: 15~20%
시연: 20%
직접 실습: 35~40%
오류 해결과 질문: 15%
문서 정리와 체크리스트: 10%
```

중요한 점은 설명 시간을 너무 길게 가져가지 않는 것입니다. 새로운 기술은 먼저 역할을 이해하고, 바로 작은 예제를 실행하면서 익히는 방식이 좋습니다.

## 프로젝트 수업일 운영 기준

미니 프로젝트나 최종 프로젝트를 진행하는 날은 구현보다 설계와 검증 시간을 충분히 확보합니다.

```text
09:00~09:30: 프로젝트 목표와 산출물 기준 확인
09:30~10:30: 요구사항 정리와 설계 문서 작성
10:30~10:40: 휴식
10:40~12:00: 기본 기능 구현 또는 도구 설정
12:00~13:00: 점심
13:00~14:30: 핵심 기능 구현
14:30~15:20: 테스트 체크리스트 작성과 실행
15:20~15:30: 휴식
15:30~16:30: 오류 해결과 보완
16:30~17:20: README, 결과 보고서, 발표 자료 정리
17:20~18:00: 시연 준비, 제출물 확인, 회고
```

프로젝트 수업에서는 결과 화면만 보는 것이 아니라 다음 항목을 함께 확인합니다.

```text
요구사항이 문서에 정리되어 있는가?
API, 화면, 데이터 저장 흐름이 서로 맞는가?
테스트 결과와 오류 처리 내용이 기록되어 있는가?
README만 보고 실행 순서를 따라갈 수 있는가?
민감정보가 제출물에 포함되지 않았는가?
```

## 8시간 수업에서 반드시 확보할 시간

하루 수업이 길어도 아래 시간은 줄이지 않는 것이 좋습니다.

```text
수업 시작 20분: 오늘 목표와 폴더 위치 확인
오전 실습 전 10분: .venv, .env, 실행 위치 확인
오후 시작 20분: 오전 실습 상태 점검
마지막 30분: 체크리스트, 제출물, 다음 수업 안내
```

설치와 환경 설정이 필요한 날에는 오전 시간을 더 넉넉히 잡습니다. 특히 Python, VS Code, Docker Desktop, Supabase, n8n, Dify처럼 계정이나 설치가 필요한 도구는 예상보다 시간이 오래 걸릴 수 있습니다.

## 시간이 부족할 때 줄일 것

시간이 부족하면 아래 항목을 먼저 줄입니다.

```text
부가 설명
선택 확장 기능
디자인 꾸미기
추가 기능 구현
심화 비교 설명
발표 자료 장식
```

## 줄이면 안 되는 것

아래 항목은 수업 품질과 안전에 직접 연결되므로 줄이지 않습니다.

```text
README 읽기
현재 폴더 위치 확인
.venv 활성화 확인
requirements.txt 설치 확인
.env와 .env.example 구분
실행 결과 확인
오류 메시지 전체 확인
API key, token, service role key 보안 확인
체크리스트와 제출물 확인
```

## 하루 마무리 체크

매일 마지막에는 아래 순서로 정리합니다.

```text
1. 오늘 실행한 폴더와 파일을 확인한다.
2. 성공한 실습과 실패한 실습을 구분한다.
3. 오류가 남아 있다면 오류 메시지를 기록한다.
4. README 또는 체크리스트에 진행 상태를 표시한다.
5. 다음 수업에서 이어서 볼 파일을 정리한다.
6. .env, API key, token이 공유 자료에 포함되지 않았는지 확인한다.
```
