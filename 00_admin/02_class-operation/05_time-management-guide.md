# 05. Time Management Guide

수업 시간을 관리하기 위한 기준입니다.

## 2시간 수업 예시

```text
00~10분: 오늘 목표와 폴더 안내
10~25분: 개념 설명
25~45분: 강사 데모
45~80분: 학생 실습
80~100분: 오류 해결과 질문
100~115분: 실습 체크리스트 확인
115~120분: 다음 차시 안내
```

## 3시간 수업 예시

```text
00~15분: 오늘 목표와 복습
15~40분: 개념 설명
40~70분: 강사 데모
70~120분: 학생 실습
120~150분: 확장 실습 또는 과제
150~170분: 오류 해결
170~180분: 정리와 제출 안내
```

## 프로젝트 수업 예시

```text
00~20분: 프로젝트 목표 확인
20~50분: 팀별 설계 문서 작성
50~100분: 구현 또는 도구 설정
100~130분: 테스트 체크리스트 작성
130~160분: 발표 자료 작성
160~180분: 중간 발표 또는 피드백
```

## 시간이 부족할 때 줄일 것

```text
부가 설명
추가 기능 구현
선택 확장 기능
디자인 꾸미기
```
