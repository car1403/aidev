## 수업 운영 원칙

```text
1. 문서를 먼저 보고, 코드는 그 다음 실행한다.
2. 먼저 한 번 시연하고, 바로 따라 한다.
3. 오류는 수업의 일부로 다룬다.
4. 모든 실습은 작은 성공 단위로 나눈다.
5. 프로젝트는 코드보다 설계를 먼저 작성한다.
```
