# Tool Spec

샘플 프로젝트에서 사용하는 Tool 함수입니다.

Tool은 LLM이 직접 처리하기 어려운 작업을 Python 코드로 분리한 것입니다. 이 샘플에서는 실제 Calendar API 대신 Mock 데이터를 사용합니다.

## extract_participants

사용자 요청에서 참석자 이름을 추출합니다.

| 항목 | 내용 |
| --- | --- |
| 입력 | 사용자 요청 문자열 |
| 출력 | 참석자 이름 목록 |
| 실패 처리 | 이름을 찾지 못하면 기본 참석자 목록을 사용합니다. |

## find_common_available_slots

참석자들이 모두 가능한 시간대를 계산합니다.

| 항목 | 내용 |
| --- | --- |
| 입력 | 참석자 목록, 회의 시간 |
| 출력 | 공통 가능 시간대 목록 |
| 실패 처리 | 공통 시간이 없으면 빈 목록을 반환합니다. |

## select_best_slot

가능한 시간대 중 첫 번째 후보를 선택합니다.

| 항목 | 내용 |
| --- | --- |
| 입력 | 공통 가능 시간대 목록 |
| 출력 | 선택된 시간대 |
| 실패 처리 | 후보가 없으면 `None` 또는 fallback 메시지로 이어집니다. |

## build_rule_based_message

OpenAI API Key가 없을 때 사용할 규칙 기반 메시지를 생성합니다.

| 항목 | 내용 |
| --- | --- |
| 입력 | 선택된 시간대, 참석자 목록 |
| 출력 | 사용자에게 보여줄 일정 제안 메시지 |
| 실패 처리 | 선택된 시간이 없으면 대체 요청 메시지를 생성합니다. |

## 팀 프로젝트에서 Tool을 설계할 때

- Tool 이름은 동사로 시작하면 이해하기 쉽습니다.
- 입력과 출력은 문서에 먼저 적은 뒤 코드로 옮깁니다.
- 외부 API를 붙이기 전에 Mock data로 먼저 테스트합니다.
- Tool 실패 결과도 State에 저장해야 디버깅하기 쉽습니다.
