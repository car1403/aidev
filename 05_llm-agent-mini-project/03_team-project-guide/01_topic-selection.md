# 01 Topic Selection

이 문서는 팀별 Agent 프로젝트 주제를 정하기 위한 가이드입니다.

이번 과정의 기준 주제는 **복합 API 연계 일정 조정 Agent**입니다. 완전히 다른 주제를 새로 만들기보다, 같은 일정 조정 구조 안에서 Tool, 데이터, 검증 기준을 다르게 설계하는 것을 권장합니다.

## 기준 주제

```text
단일 Agent 추론 및 AI 오케스트레이션
-> 복합 API 연계 일정 조정 Agent
```

## 작성 항목

- 프로젝트 이름
- 팀 구성
- 주요 사용자
- 사용자 요청 예시 3개
- Agent가 제공할 최종 결과
- 필요한 Tool 목록
- Mock data 목록
- 선택 확장 API 목록
- Session Memory 필요 여부
- Long-term Memory 필요 여부
- 실패 또는 fallback 시나리오

## 추천 변형 주제

- 팀 회의 일정 조정 Agent
- 상담 일정 조정 Agent
- 병원 또는 상담 예약 후보 시간 추천 Agent
- 여러 참석자의 가능한 시간을 비교하는 회의 조율 Agent
- 일정 충돌을 감지하고 대체 시간을 제안하는 업무 비서 Agent

## 좋은 주제 조건

- Mock data로 먼저 구현할 수 있습니다.
- Tool이 2개 이상 필요합니다.
- Decision Node가 필요한 이유가 있습니다.
- 실패하거나 애매한 요청에 대한 fallback이 있습니다.
- Streamlit 화면으로 실행 결과와 State를 보여주기 쉽습니다.
- 테스트 결과를 숫자 또는 체크리스트로 비교할 수 있습니다.

## 주제 선정 양식

```text
프로젝트명:

사용자 요청 예시 1:
사용자 요청 예시 2:
사용자 요청 예시 3:

필요한 Tool:
필요한 State 필드:
분기 조건:
fallback 조건:
테스트할 오류 유형:
최종 시연 시나리오:
```
