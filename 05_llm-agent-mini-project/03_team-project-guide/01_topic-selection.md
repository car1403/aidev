# 01 Topic Selection

팀별 Agent 프로젝트 주제를 정합니다.

이번 과정의 기준 주제는 **복합 API 연계 일정 조정 에이전트**입니다. 팀별로 완전히 다른 주제를 만들기보다, 같은 일정 조정 구조 안에서 사용하는 도구, 데이터, 검증 기준을 다르게 설계하는 것을 권장합니다.

## 기준 주제

```text
단일 에이전트 추론 및 AI 오케스트레이션
-> 복합 API 연계 일정 조정 에이전트
```

## 작성 항목

- 프로젝트 이름
- 대상 사용자
- 사용자가 입력할 일정 요청 예시 3개
- 에이전트가 제공할 최종 결과
- 필요한 Tool 목록
- 외부 API 또는 Mock data 목록
- Session Memory 필요 여부
- Long-term Memory 필요 여부
- 실패 또는 fallback 시나리오

## 추천 변형 주제

- 팀 회의 일정 조정 에이전트
- 수강생 상담 일정 조정 에이전트
- 병원/상담 예약 후보 시간 추천 에이전트
- 여러 참석자의 가능 시간을 비교하는 회의 조율 에이전트
- 일정 충돌을 감지하고 대체 시간을 제안하는 업무 비서 에이전트

## 좋은 주제 조건

- Mock data로 먼저 구현 가능하다.
- Tool이 2개 이상 필요하다.
- Decision Node가 필요한 이유가 있다.
- 실패하거나 애매한 요청에 대한 fallback이 있다.
- Streamlit 화면으로 실행 결과와 State를 보여주기 쉽다.
- 시험 결과를 수치로 비교할 수 있다.

## 주제 선정 양식

```text
프로젝트명:
팀명:
대상 사용자:
사용자 요청 예시 1:
사용자 요청 예시 2:
사용자 요청 예시 3:
필요한 Tool:
필요한 상태 필드:
분기 조건:
fallback 조건:
시험할 오류 유형:
최종 시연 시나리오:
```
