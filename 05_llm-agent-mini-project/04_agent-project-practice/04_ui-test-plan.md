# 04 UI Test Plan

이 문서는 Streamlit 화면 테스트 계획입니다.

UI는 단순히 예쁘게 보이는 화면이 아니라 Agent가 어떤 상태로 실행되는지 확인하는 디버깅 도구이기도 합니다.

## 화면 요소

- 요청 입력창
- 실행 버튼
- 최종 답변 영역
- Agent State 디버그 영역
- Tool 호출 이력 영역
- 오류 메시지 영역

## 테스트 입력

| 유형 | 예시 | 기대 결과 |
| --- | --- | --- |
| 정상 요청 | `민수와 지영이 가능한 30분 회의 시간을 찾아줘` | 가능한 시간 제안 |
| 정보 부족 요청 | `회의 잡아줘` | 추가 정보 요청 또는 fallback |
| Tool 결과 없음 | `모두 가능한 시간이 없는 일정` | 대체 시간 제안 |
| 너무 긴 요청 | 여러 조건이 섞인 요청 | 핵심 조건 요약 후 처리 |
| 애매한 참석자 | 이름이 누락된 요청 | 참석자 확인 요청 |

## 확인 기준

- 입력값이 State에 저장되는가?
- Tool 호출 이력이 화면에서 확인되는가?
- 최종 답변이 이해하기 쉽게 표시되는가?
- 오류 메시지가 개발자용이 아니라 사용자에게 이해 가능한 문장인가?
- 디버그 State가 너무 길면 접거나 요약할 수 있는가?
