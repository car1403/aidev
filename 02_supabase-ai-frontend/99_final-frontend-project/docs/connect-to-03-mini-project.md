# 03 미니 프로젝트로 연결하기

`99_final-frontend-project`는 `03_supabase-ai-mini-project`로 넘어가기 전에 프론트엔드 통합 감각을 익히는 작은 예제입니다.

## 02에서 만든 것

```text
Streamlit 화면
-> FastAPI 백엔드 호출
-> 챗봇 응답 표시
-> 대화 이력 조회
-> 서비스 로그 조회
-> 무료 배포 흐름 확인
```

## 03에서 확장할 것

03에서는 같은 흐름을 실제 프로젝트에 가깝게 확장합니다.

| 02 최종 예제 | 03 미니 프로젝트 확장 |
| --- | --- |
| 메모리 기반 대화 이력 | Supabase `conversations`, `messages` 테이블 저장 |
| 메모리 기반 서비스 로그 | Supabase `service_logs` 테이블 저장 |
| mock AI 응답 | Gemini API 기반 실제 응답 생성 |
| 일반 POST 응답 | SSE 기반 실시간 응답 스트리밍 |
| 간단한 로그 화면 | 실시간 로그 대시보드 |
| 간단한 배포 점검 | API 설계 문서, 화면 설계서, DB 설계서, 대시보드 산출물 |

## 연결 구조 요약

```text
02:
Streamlit -> FastAPI -> 메모리 저장

03:
Streamlit -> FastAPI -> Supabase
                     -> Gemini API
                     -> SSE 스트리밍
                     -> 로그/피드백 대시보드
```

## 03으로 넘어가기 전 질문

1. 지금 예제에서 대화 이력이 서버 재시작 후 사라지는 이유는 무엇인가요?
2. 이 데이터를 Supabase에 저장하려면 어떤 테이블이 필요할까요?
3. 서비스 로그에는 어떤 정보가 남아야 운영에 도움이 될까요?
4. 사용자가 응답 품질에 대한 피드백을 남기려면 어떤 컬럼이 필요할까요?
5. AI 응답이 한 번에 늦게 나타나는 문제를 SSE로 어떻게 개선할 수 있을까요?

## 03에서 만들 산출물

- API 설계 문서
- 화면 설계서
- 데이터베이스 설계서
- 대시보드 구현 결과물
- 배포 선택 사항과 실행 결과 정리
