# UI Review Checklist

## 화면 흐름

- 사용자가 무엇을 먼저 입력해야 하는지 명확한가?
- 버튼을 누른 뒤 결과가 어디에 표시되는지 알 수 있는가?
- 오류 메시지가 사용자가 이해할 수 있는 문장인가?
- 입력 영역과 결과 영역이 구분되어 있는가?

## 코드 구조

- 한 파일이 너무 길지 않은가?
- 반복되는 UI 코드가 함수로 분리되어 있는가?
- API 호출 코드가 화면 출력 코드와 섞여 있지 않은가?
- session_state 초기화가 파일 상단에서 일관되게 처리되는가?

## 데이터와 상태

- 빈 입력값을 처리하는가?
- API 실패 상황을 처리하는가?
- 로그인 토큰을 화면에 그대로 노출하지 않는가?
- 캐시가 사용자별 데이터에 잘못 적용되지 않는가?

## 제출 전 확인

- `streamlit run`으로 실행되는가?
- Python 문법 오류가 없는가?
- README의 실행 명령과 실제 파일 경로가 일치하는가?
- 수정한 이유를 본인이 설명할 수 있는가?
