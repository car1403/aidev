# 01. Course Roadmap

이 문서는 01~08 과정 전체 로드맵입니다.

## 전체 로드맵

```text
01_python-git-foundation
-> 02_supabase-ai-backend
-> 03_supabase-ai-frontend
-> 04_supabase-ai-mini-project
-> 05_llm-agent-orchestration
-> 06_llm-agent-mini-project
-> 07_multi-agent-service-ops
-> 08_multi-agent-service-mini-project
```

## 단계별 의미

| 단계 | 과정 | 의미 |
| --- | --- | --- |
| 01 | Python/Git Foundation | Python 기본기와 Git/GitHub 협업 기초를 먼저 분리해서 학습합니다. |
| 02 | Supabase AI Backend | FastAPI로 백엔드를 만들고 Supabase에 데이터를 저장합니다. |
| 03 | Supabase AI Frontend | Streamlit으로 화면을 만들고 백엔드 API를 호출합니다. |
| 04 | Supabase AI Mini Project | 백엔드, 프론트엔드, Supabase를 연결한 작은 AI 서비스를 만듭니다. |
| 05 | LLM Agent Orchestration | Docker 기반으로 LLM, Tool, Memory, RAG, Agent 흐름을 배웁니다. |
| 06 | LLM Agent Mini Project | 05에서 배운 Agent 구조로 단일 Agent 미니 프로젝트를 만듭니다. |
| 07 | Multi-Agent Service Ops | Docker Compose, AWS, GitHub Actions, 운영/보안/모니터링을 배웁니다. |
| 08 | Multi-Agent Service Mini Project | 운영 관점의 Multi-Agent Auto Healing 미니 프로젝트를 만듭니다. |

## 핵심 흐름

```text
Python 기초
-> Git/GitHub
-> API 서버
-> 화면 UI
-> Supabase 저장
-> AI 응답
-> 실시간 스트리밍
-> Agent 구조
-> Docker 기반 운영
-> Multi-Agent Auto Healing 프로젝트
```

## 한 줄 요약

```text
01: Python/Git 기반을 먼저 다진다.
02~04: 작은 AI 서비스를 직접 만든다.
05~06: AI가 Tool과 Memory를 사용하는 Agent 구조를 배운다.
07~08: 서비스를 운영 가능한 형태로 만들고 Multi-Agent 미니 프로젝트로 완성한다.
```
