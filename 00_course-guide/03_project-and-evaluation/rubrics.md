# Rubrics

평가는 선택 확장 기능보다 필수 산출물과 핵심 흐름의 설명 가능성을 우선합니다.

## 공통 평가 기준

| 항목 | 확인 내용 |
| --- | --- |
| 문제 정의 | 사용자, 문제, 입력/출력이 명확한가 |
| 구조 설계 | API, 화면, 데이터, Agent 흐름이 논리적인가 |
| 구현 결과 | README 기준으로 실행 또는 시연 가능한가 |
| 오류 처리 | 실패 상황과 fallback 또는 대응 방법이 있는가 |
| 보안 | `.env`, API Key, token을 제출하지 않았는가 |
| 기록 | 테스트 결과와 개선 과정이 문서화되어 있는가 |

## 프로젝트별 중점

| 과정 | 평가 중점 |
| --- | --- |
| 04 | API 설계, 화면 설계, Supabase 스키마, 대시보드 동작 |
| 06 | Agent 상태 흐름, Tool 선택 기준, 오류 감지와 재시도/대체 전략 |
| 08 | Multi-Agent 역할 분담, handoff, 장애 감지/복구, 파이프라인 검증 |

선택 보조 산출물은 가산 또는 설명 보조로만 사용하고, 필수 제출 기준을 대체하지 않습니다.
