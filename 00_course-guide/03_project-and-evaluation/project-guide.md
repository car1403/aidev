# Project Guide

프로젝트 과정은 `04`, `06`, `08`입니다.

| 과정 | 프로젝트 성격 | 필수 산출물 |
| --- | --- | --- |
| 04 Supabase AI Mini Project | FastAPI, Streamlit, Supabase를 연결한 AI 서비스 | API 설계 문서, 화면 설계서, DB 스키마 설계서, 대시보드 구현 결과물 |
| 06 LLM Agent Mini Project | 단일 Agent의 판단, 도구 선택, 오류 감지와 검증 | Agent 아키텍처 설계서, Agent 시험 결과 보고서 |
| 08 Multi-Agent Service Mini Project | 여러 Agent가 협업하는 운영/복구 서비스 | 멀티 에이전트 아키텍처 설계서, 배포 및 장애 복구 보고서, 파이프라인 구현 결과 보고서 |

## 주제 선정 기준

```text
[ ] 사용자가 명확한가?
[ ] 해결하려는 문제가 구체적인가?
[ ] 입력과 출력이 설명되는가?
[ ] 수업 시간 안에 핵심 기능을 구현할 수 있는가?
[ ] 필수 산출물을 먼저 완성할 수 있는가?
```

주제가 너무 크면 사용자 1명, 핵심 API 2~3개, 핵심 화면 1~2개로 줄입니다. 실시간, 배포, 자동화, 고급 보안은 선택 확장으로 둡니다.
