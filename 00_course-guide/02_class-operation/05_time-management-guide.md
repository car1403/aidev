# 05. Time Management Guide

하루 8시간 진행을 기준으로 시간을 관리하기 위한 안내입니다.

실제 진행 속도는 설치 상황, 오류 해결, 프로젝트 난이도에 따라 달라질 수 있습니다. 다만 기본 흐름은 아래 8시간 구조를 기준으로 잡습니다.

## 하루 8시간 기본 구조

```text
09:00~09:20 오늘 목표와 폴더 위치 확인
09:20~10:00 핵심 개념 설명
10:00~10:50 첫 번째 시연
10:50~11:00 휴식
11:00~12:00 기본 실습
12:00~13:00 점심
13:00~13:20 오전 실습 상태 점검
13:20~14:20 두 번째 시연 또는 확장 개념 설명
14:20~15:20 확장 실습
15:20~15:30 휴식
15:30~16:30 Lab 또는 Assignment 진행
16:30~17:20 오류 해결, 코드 리뷰, 문서 정리
17:20~17:50 체크리스트 확인과 산출물 정리
17:50~18:00 다음 과정 연결 안내
```

## 일반 실습 비율

```text
개념 설명: 15~20%
시연: 20%
직접 실습: 35~40%
오류 해결과 질문: 15%
문서 정리와 체크리스트: 10%
```

## 프로젝트 진행일 기준

```text
09:00~09:30 프로젝트 목표와 산출물 기준 확인
09:30~10:30 요구사항 정리와 설계 문서 작성
10:30~10:40 휴식
10:40~12:00 기본 기능 구현 또는 도구 설정
12:00~13:00 점심
13:00~14:30 핵심 기능 구현
14:30~15:20 테스트 체크리스트 작성과 실행
15:20~15:30 휴식
15:30~16:30 오류 해결과 보완
16:30~17:20 README, 결과 보고서, 발표 자료 정리
17:20~18:00 시연 준비, 제출물 확인, 회고
```

## 시간을 반드시 확보할 부분

```text
수업 시작 20분: 오늘 목표와 폴더 위치 확인
실전 실습 전 10분: .venv, .env, 실행 위치 확인
오후 시작 20분: 오전 실습 상태 점검
마지막 30분: 체크리스트, 제출물, 다음 과정 안내
```

설치와 환경 설정이 필요한 날에는 오전 시간을 더 넉넉히 잡습니다. 특히 Python, VS Code, GitHub 계정, GitHub Copilot Chat, Docker Desktop, Supabase처럼 계정이나 설치가 필요한 도구는 예상보다 시간이 오래 걸릴 수 있습니다.

## 시간이 부족할 때 줄일 항목

```text
부가 설명
선택 확장 기능
디자인 꾸미기
추가 기능 구현
발표 자료 장식
```

## 줄이면 안 되는 항목

```text
README 읽기
현재 폴더 위치 확인
.venv 활성화 확인
requirements.txt 설치 확인
.env와 .env.example 구분
실행 결과 확인
오류 메시지 전체 확인
API key, token, service role key 보안 확인
체크리스트와 제출물 확인
```
