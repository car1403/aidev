# -----------------------------------------------------------------------------
# Streamlit 프론트엔드 화면 구성 패키지
# -----------------------------------------------------------------------------

# Python 코드만으로 웹 화면을 빠르게 만들기 위한 UI 프레임워크입니다.
# 이 과정의 중심 프론트엔드 도구입니다.
# 사용 위치:
# - 01_streamlit-basic: Streamlit 실행, 텍스트, 버튼, 입력 컴포넌트 기초
# - 02_streamlit-ui-components: 입력폼, 선택박스, 파일 업로드, 표와 차트 화면 구성
# - 04_ai-chatbot-interface: 챗봇 입력 화면과 응답 표시 화면
# - 05_state-session-and-data: st.session_state 기반 대화 상태 관리
# - 90_ai-assisted-ui-review-and-debugging: Streamlit UI 오류 점검
# - 99_final-frontend-project: 최종 프론트엔드 예제와 배포 실습
streamlit==1.41.1


# -----------------------------------------------------------------------------
# 표, 데이터프레임, 차트 표시 패키지
# -----------------------------------------------------------------------------

# 표 형태의 데이터를 다루고 화면에 보기 좋게 표시하기 위한 패키지입니다.
# Streamlit의 st.dataframe, st.table, st.bar_chart 등과 함께 자주 사용합니다.
# 사용 위치:
# - 02_streamlit-ui-components: 표와 차트 표시
# - 05_state-session-and-data: 대화 이력과 사용자 상태 데이터 표시
# - 99_final-frontend-project: 대화 이력, 서비스 로그를 표와 차트로 표시
pandas==2.2.3


# -----------------------------------------------------------------------------
# FastAPI 백엔드 호출 패키지
# -----------------------------------------------------------------------------

# Streamlit 프론트엔드에서 FastAPI 백엔드 API를 호출하기 위한 HTTP 클라이언트입니다.
# 프론트엔드는 직접 데이터베이스에 접근하기보다 백엔드 API를 호출하는 구조로 학습합니다.
# 사용 위치:
# - 03_api-integration: Streamlit에서 sample backend 또는 01 백엔드 API 호출
# - 04_ai-chatbot-interface: mock 챗봇 API 또는 선택형 Gemini 연동 API 호출
# - 05_state-session-and-data: 사용자별 대화 이력, 서비스 로그 API 호출 흐름 이해
# - 99_final-frontend-project: /health, /api/chat, /api/chats, /api/logs 호출
httpx==0.28.1


# -----------------------------------------------------------------------------
# 환경 변수 관리 패키지
# -----------------------------------------------------------------------------

# .env 파일에 적은 환경 변수를 Python 코드에서 읽어 오기 위한 패키지입니다.
# 프론트엔드에서는 주로 API_BASE_URL처럼 백엔드 주소를 읽을 때 사용합니다.
# 사용 위치:
# - 03_api-integration: API_BASE_URL로 백엔드 주소 관리
# - 04_ai-chatbot-interface: 선택형 API 연결 주소 관리
# - 05_state-session-and-data: API 호출 주소와 실행 환경 분리
# - 99_final-frontend-project: 프로젝트 폴더 안의 .env에서 API_BASE_URL 읽기
#
# 주의:
# - 프론트엔드 .env에는 공개되어도 되는 설정만 넣습니다.
# - Supabase service role key, Upstash token, LLM API key는 프론트엔드에 넣지 않습니다.
python-dotenv==1.0.1


# -----------------------------------------------------------------------------
# 선택형 Gemini SDK 실습 패키지
# -----------------------------------------------------------------------------

# Gemini API를 Python SDK 방식으로 호출하기 위한 Google Gen AI SDK입니다.
# 02 과정에서는 실제 LLM 호출을 깊게 다루기보다,
# 02_supabase-ai-backend에서 배운 Gemini 호출 흐름을 프론트엔드 화면과 가볍게 연결해 봅니다.
# 기본 예제 모델은 gemini-2.5-flash-lite를 기준으로 안내합니다.
# 사용 위치:
# - 04_ai-chatbot-interface: mock 응답 이후 선택형 Gemini 호출 예제
# - 90_ai-assisted-ui-review-and-debugging: LLM 호출 오류 메시지와 환경 변수 점검 예시
#
# 주의:
# - Gemini API key는 프론트엔드 화면 코드에 직접 넣지 않는 것이 원칙입니다.
# - 실제 서비스 구조에서는 FastAPI 백엔드가 Gemini API를 호출하고,
#   Streamlit은 백엔드 API만 호출하는 구조가 더 안전합니다.
google-genai==1.2.0
