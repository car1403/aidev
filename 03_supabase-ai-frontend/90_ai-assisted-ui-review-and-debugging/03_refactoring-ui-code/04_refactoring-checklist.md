# Refactoring Checklist

## 함수 분리

- 입력 검증을 함수로 분리했는가?
- 응답 생성 로직을 함수로 분리했는가?
- API 호출을 함수로 분리했는가?
- 화면 출력 함수와 데이터 처리 함수가 구분되어 있는가?
- Authorization header 생성 함수를 분리했는가?
- 오류 메시지를 표시하는 흐름이 반복된다면 함수로 분리했는가?

## 이름

- 함수 이름이 역할을 설명하는가?
- 변수 이름이 데이터의 의미를 설명하는가?
- 상수 이름이 설정값임을 드러내는가?

## 검증

- 리팩토링 전후 실행 결과가 같은가?
- 오류 처리 로직이 유지되는가?
- README의 실행 명령이 여전히 맞는가?
- API 호출 경로와 응답 키 이름이 바뀌지 않았는가?
- 프론트엔드에 비밀 키를 추가하지 않았는가?
