# Mini Project Deliverables Guide

이 문서는 `03_supabase-ai-mini-project`의 필수 산출물을 작성할 때 참고하는 가이드입니다.

## 산출물 1. API 설계 문서

작성 파일:

```text
docs/api-spec.md
```

점검 질문:

- 엔드포인트 URL이 리소스 중심으로 일관되게 명명되었나요?
- `GET`, `POST`, `PUT`, `DELETE`가 의미에 맞게 사용되었나요?
- 에러 응답 형식에 HTTP Status Code, 에러 코드, 메시지, 상세 정보가 포함되었나요?
- 4xx/5xx 예외 상황별 처리 규칙이 문서화되었나요?
- Request/Response Pydantic 모델에 필수/선택 필드, 타입 힌트, 예시값이 있나요?
- 중첩 JSON은 nested 모델 또는 명확한 JSON 구조로 표현되었나요?

## 산출물 2. 화면 설계서

작성 파일:

```text
docs/ui-design.md
```

점검 질문:

- 메인, 상세, 입력, 설정, 오류 화면의 와이어프레임이 있나요?
- 정보 계층 구조(IA)가 논리적인가요?
- 버튼, 입력폼, 테이블, 차트의 스타일과 배치 기준이 일관적인가요?
- 클릭, 호버, 드래그, 스크롤, 필터, 새로고침에 대한 시스템 반응이 정의되었나요?
- 화면별 API 호출과 오류 표시 방식이 정리되었나요?

## 산출물 3. 데이터베이스 설계서

작성 파일:

```text
docs/supabase-schema.md
```

점검 질문:

- 테이블이 3정규화 기준으로 설계되었나요?
- 데이터 중복과 삽입/갱신/삭제 이상현상이 줄어들도록 설계되었나요?
- 논리 ERD에 엔티티, 속성, 관계, 카디널리티가 포함되었나요?
- 물리 ERD에 테이블명, 컬럼, PK/FK, 인덱스가 포함되었나요?
- 모든 컬럼의 타입, 길이, 제약조건, 기본값, 코멘트가 작성되었나요?

## 산출물 4. 대시보드 구현 결과물

작성 파일:

```text
docs/dashboard-result.md
frontend/app.py
```

점검 질문:

- 대시보드가 로컬 또는 배포 환경에서 실행되나요?
- 로그 수집, 조회, 시각화가 시연 가능한가요?
- 사용자 피드백 데이터가 저장되거나 대시보드에 반영되나요?
- SSE 또는 자동 새로고침 기반 실시간 표시 방식이 설명되어 있나요?
- AI 답변 품질 개선 방향이 문서화되어 있나요?
