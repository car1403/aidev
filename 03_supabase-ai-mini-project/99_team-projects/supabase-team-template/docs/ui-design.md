# UI Design Document

이 문서는 Streamlit 화면 설계서입니다.

## 1. 화면 목록

| Screen | Purpose | Main Components | API Call |
| --- | --- | --- | --- |
| 메인 대시보드 | 핵심 지표와 최근 로그 확인 | metric, chart, table | `GET /api/dashboard/summary`, `GET /api/logs` |
| 로그 입력 화면 | 테스트 로그 또는 사용자 질문 등록 | form, text_area, selectbox, button | `POST /api/logs`, `POST /api/chat` |
| 로그 상세 화면 | 선택한 로그 상세 확인 | table, json viewer, feedback form | `GET /api/logs/{log_id}` |
| 피드백 화면 | AI 응답 품질 평가 | radio, slider, text_area | `POST /api/feedback` |
| 설정 화면 | API 주소, 필터 기본값, 새로고침 방식 확인 | text_input, selectbox, checkbox | 환경변수 또는 local state |
| 오류 화면 | 오류 또는 빈 데이터 안내 | warning, error, retry button | 상황별 |

## 2. Information Architecture

```text
메인 대시보드
|-- 요약 지표
|-- 필터 영역
|-- 로그 목록
|-- 로그 상세
`-- 피드백 입력
```

## 3. Wireframe

```text
-------------------------------------------------
실시간 로그 대시보드
-------------------------------------------------
[전체 로그] [오류 로그] [평균 처리 시간] [평균 피드백 점수]

필터: [기간] [상태] [로그 유형] [새로고침]

차트 영역
-------------------------------------------------
로그 목록 테이블
-------------------------------------------------
선택한 로그 상세 / AI 응답 / 사용자 피드백
-------------------------------------------------
```

## 4. User Actions

| Action | System Response |
| --- | --- |
| 새로고침 클릭 | 최신 로그를 다시 조회하고 표와 차트를 갱신 |
| 필터 변경 | 선택 조건으로 로그 목록을 다시 조회 |
| 로그 행 클릭 | 상세 영역에 로그 본문과 metadata 표시 |
| 질문 입력 후 제출 | AI 응답 API 호출, 응답과 로그 저장 |
| 피드백 제출 | feedback 테이블에 점수와 의견 저장 |
| SSE 시작 | 응답 chunk를 화면에 누적 표시하고 완료 후 최종 응답 저장 |
| API 오류 발생 | 사용자가 다음 행동을 알 수 있는 오류 메시지 표시 |

## 5. Design Rules

- 버튼, 입력폼, 테이블, 차트 스타일은 한 화면 안에서 일관되게 유지합니다.
- 오류 메시지는 사용자가 다음 행동을 알 수 있게 작성합니다.
- 빈 데이터 화면은 단순히 비워두지 않고 안내 문구를 표시합니다.
- 로딩 중에는 spinner 또는 status 메시지를 표시합니다.

## 6. Detailed UI Review Checklist

- [ ] 메인, 상세, 입력, 설정, 오류 화면이 모두 정의되어 있습니다.
- [ ] 화면별 와이어프레임이 누락 없이 작성되어 있습니다.
- [ ] 정보 계층 구조(IA)가 논리적입니다.
- [ ] 버튼, 입력폼, 테이블, 차트의 스타일, 색상, 폰트, 여백이 일관적입니다.
- [ ] 클릭, 호버, 드래그, 스크롤, 필터 적용, 새로고침 액션에 대한 반응이 정의되어 있습니다.
- [ ] API 호출 중 로딩 상태가 표시됩니다.
- [ ] API 실패, 인증 실패, 빈 데이터 상황에 대한 화면 메시지가 정의되어 있습니다.
- [ ] 사용자 피드백 입력 후 대시보드 지표가 어떻게 바뀌는지 설명되어 있습니다.
