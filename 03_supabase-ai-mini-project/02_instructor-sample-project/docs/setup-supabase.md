# Setup Supabase

Supabase 학습 로그 샘플을 실행하기 위한 테이블 준비 문서입니다.

이 샘플은 `learning_logs` 테이블을 사용합니다. 먼저 Supabase 프로젝트와 테이블을 준비한 뒤 FastAPI 백엔드와 Streamlit 프론트엔드를 실행합니다.

## 1. Supabase 프로젝트 준비

Supabase Dashboard에서 프로젝트를 선택하거나 새로 만듭니다.

필요한 값은 아래 메뉴에서 확인합니다.

```text
Project Settings
-> API
```

확인할 값:

```text
SUPABASE_URL
SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY
```

`SUPABASE_SERVICE_ROLE_KEY`는 강한 권한을 가진 서버용 key입니다. Streamlit 프론트엔드 코드, 화면, README, 과제 문서에 적지 않습니다.

## 2. 테이블 만들기

Supabase SQL Editor에서 다음 SQL을 실행합니다.

```sql
create table if not exists public.learning_logs (
  id bigint generated by default as identity primary key,
  learner_name text not null,
  topic text not null,
  minutes integer not null check (minutes > 0),
  status text not null default 'created',
  memo text default '',
  created_at timestamptz not null default now()
);
```

## 3. 샘플 데이터 넣기

샘플 데이터를 넣으면 Streamlit 화면에서 바로 조회 결과를 확인할 수 있습니다.

```sql
insert into public.learning_logs (learner_name, topic, minutes, status, memo)
values
  ('Kim', 'Supabase Mini Project', 60, 'done', 'Supabase table is ready.'),
  ('Lee', 'FastAPI Integration', 45, 'doing', 'Health check API is ready.'),
  ('Park', 'Streamlit Dashboard', 50, 'created', 'Dashboard screen will be connected.');
```

이미 샘플 데이터를 넣었다면 중복으로 넣지 않아도 됩니다.

## 4. .env 파일 준비

이 샘플은 `02_instructor-sample-project` 폴더 안의 `.env`를 사용하지 않습니다. 03 과정 최상위 `.env`를 사용합니다.

```text
C:\aidev\03_supabase-ai-mini-project\.env
```

아직 `.env`가 없다면 최상위 폴더에서 복사합니다.

```powershell
cd C:\aidev\03_supabase-ai-mini-project
Copy-Item .env.example .env
```

필수 값:

```env
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_ANON_KEY=your-supabase-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-supabase-service-role-key
API_BASE_URL=http://127.0.0.1:8000
```

Gemini와 OpenAI 값은 AI 응답 생성 실습에서 사용합니다. 이 샘플은 먼저 Supabase 저장과 조회가 되는지 확인하는 프로젝트입니다.

## 5. RLS 참고

이 샘플의 FastAPI 백엔드는 `SUPABASE_SERVICE_ROLE_KEY`가 있으면 서버 내부에서 우선 사용하고, 값이 없으면 `SUPABASE_ANON_KEY`로 기본 연결을 확인합니다.

팀 프로젝트에서는 다음 내용을 데이터베이스 설계서에 정리합니다.

```text
1. 어떤 API가 어떤 테이블에 접근하는가?
2. service role key가 필요한 작업은 무엇인가?
3. anon key와 RLS 정책으로 처리할 작업은 무엇인가?
4. 사용자가 자기 데이터만 조회해야 하는 경우 어떤 정책이 필요한가?
```
