# 04_service-log-and-feedback

이 단계에서는 서비스 실행 기록과 사용자 피드백을 저장하는 흐름을 정리합니다.

서비스 로그는 단순한 개발 기록이 아니라, 프로젝트가 실제 서비스처럼 운영될 수 있는지 보여주는 증거입니다. 어떤 API가 호출되었는지, 오류가 있었는지, 사용자가 어떤 피드백을 남겼는지를 기록하면 결과 발표와 개선 작업이 훨씬 쉬워집니다.

## 학습 목표

- 서비스 로그에 어떤 정보를 남길지 정할 수 있습니다.
- 사용자 피드백을 데이터로 저장할 수 있습니다.
- 로그와 피드백을 화면에서 조회할 수 있습니다.
- 프로젝트 산출물에 품질 개선 근거를 정리할 수 있습니다.
