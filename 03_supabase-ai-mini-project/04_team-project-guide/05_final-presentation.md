# 05 Final Presentation

최종 발표는 구현 결과를 보여주는 시간입니다.

발표에서는 코드 전체를 설명하기보다, 어떤 문제를 해결하려 했고 어떤 구조로 만들었으며 실제로 어떻게 동작하는지를 보여주는 것이 중요합니다.

## 발표 구성

권장 발표 순서는 다음과 같습니다.

```text
1. 프로젝트 주제와 대상 사용자
2. 전체 아키텍처
3. Supabase 테이블 구조
4. FastAPI API 구조
5. Streamlit 화면 흐름
6. 로그/피드백/AI 응답 흐름
7. 실행 시연
8. 어려웠던 점과 개선 방향
```

## 시연 흐름 예시

```text
FastAPI 실행
-> Streamlit 실행
-> 사용자 입력 또는 로그 등록
-> Supabase 저장 확인
-> 대시보드 조회
-> 피드백 입력
-> 오류 또는 빈 데이터 처리 확인
```

SSE를 적용했다면 아래 흐름도 보여줍니다.

```text
사용자 질문 입력
-> AI 응답 chunk가 화면에 누적 표시
-> 최종 assistant 응답이 Supabase에 저장
```

## 발표 자료에 포함할 내용

- 프로젝트 한 줄 설명
- 아키텍처 다이어그램
- API 목록
- 주요 테이블 ERD
- 화면 캡처
- 테스트 결과
- 선택 확장 기능
- 향후 개선 방향

## 발표 체크리스트

- [ ] 실행 순서를 3분 안에 설명할 수 있다.
- [ ] API와 DB, UI가 어떻게 연결되는지 설명할 수 있다.
- [ ] 실제 화면 시연이 가능하다.
- [ ] 민감 정보가 화면에 노출되지 않는다.
- [ ] 팀원별 담당 작업을 설명할 수 있다.
