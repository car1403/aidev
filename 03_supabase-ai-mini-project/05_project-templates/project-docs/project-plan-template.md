# Project Plan Template

## 1. 프로젝트 이름

```text
예: 학습 기록 기반 AI 피드백 로그 대시보드
```

## 2. 프로젝트 한 줄 설명

```text
누가, 어떤 문제를 해결하기 위해, 어떤 데이터를 저장하고 보여주는지 한 문장으로 작성합니다.
```

## 3. 대상 사용자

```text
예: 온라인 강의를 듣는 학습자, 수업 운영자, 고객 문의 담당자
```

## 4. 핵심 문제

```text
현재 어떤 정보가 흩어져 있는가?
어떤 로그를 모으면 도움이 되는가?
대시보드가 어떤 판단을 도와주는가?
```

## 5. 핵심 기능

| 번호 | 기능 | 설명 | 필수 여부 |
| --- | --- | --- | --- |
| 1 | 로그 등록 | 사용자 요청 또는 이벤트를 저장 | 필수 |
| 2 | 로그 조회 | 저장된 로그 목록과 상세 확인 | 필수 |
| 3 | 대시보드 | 로그 통계와 상태를 화면에 표시 | 필수 |
| 4 | 피드백 저장 | 응답 품질 평가 저장 | 권장 |
| 5 | SSE 스트리밍 | AI 응답 생성 과정을 실시간 표시 | 선택 |

## 6. 사용 기술

```text
Backend: FastAPI
Frontend: Streamlit
Database: Supabase
AI API: Gemini 기본, OpenAI 선택 비교
Deployment: 선택 확장
```

## 7. 역할 분담

| 역할 | 담당자 | 주요 작업 |
| --- | --- | --- |
| Backend/API |  | FastAPI API 구현 |
| Supabase/DB |  | 테이블 설계와 SQL 작성 |
| Frontend/UI |  | Streamlit 화면 구현 |
| AI/Quality |  | Gemini 연동, 응답 품질, 피드백 |
| Docs/Presentation |  | 문서와 발표 자료 |

## 8. 일정

| 날짜 | 목표 | 완료 기준 |
| --- | --- | --- |
| Day 1 | 설계와 첫 실행 | FastAPI, Streamlit, Supabase 연결 |
| Day 2 | 핵심 기능 구현 | 저장, 조회, 대시보드 표시 |
| Day 3 | 문서와 발표 | 산출물 정리와 시연 |
