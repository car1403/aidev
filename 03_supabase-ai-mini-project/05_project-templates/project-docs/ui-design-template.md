# UI Design Template

Streamlit 화면 설계서 템플릿입니다.

## 1. 전체 화면 구성

```text
상단:
프로젝트 제목, 현재 실행 상태

왼쪽 사이드바:
필터, 사용자 선택, 새로고침 버튼

본문:
로그 목록, 통계 지표, 차트, 상세 정보

하단:
피드백 입력, 오류 메시지, 안내 문구
```

## 2. 화면 목록

| 화면 | 목적 | 주요 컴포넌트 |
| --- | --- | --- |
| 메인 대시보드 | 전체 로그와 지표 확인 | metric, dataframe, chart |
| 로그 입력 | 새 로그 또는 질문 입력 | form, text_input, button |
| 로그 상세 | 단일 로그 상세 확인 | table, json, text_area |
| 피드백 입력 | 응답 평가 저장 | slider, text_area, button |
| 오류 안내 | API/DB 오류 안내 | error, warning |

## 3. 사용자 액션 흐름

```text
1. 사용자가 질문 또는 로그를 입력합니다.
2. 저장 버튼을 누릅니다.
3. Streamlit이 FastAPI API를 호출합니다.
4. FastAPI가 Supabase에 데이터를 저장합니다.
5. Streamlit이 결과 메시지를 표시합니다.
6. 사용자가 대시보드에서 저장된 데이터를 확인합니다.
```

## 4. 상태 메시지 기준

| 상황 | 화면 표시 |
| --- | --- |
| API 호출 중 | `st.spinner()` |
| 저장 성공 | `st.success()` |
| 입력 누락 | `st.warning()` |
| 서버 오류 | `st.error()` |
| 데이터 없음 | `st.info()` |

## 5. 디자인 기준

- 화면에 너무 많은 정보를 한 번에 넣지 않습니다.
- 버튼을 누른 뒤 결과 메시지를 반드시 표시합니다.
- 표에는 핵심 컬럼만 먼저 보여줍니다.
- 상세 정보는 펼침 영역 또는 별도 영역에 표시합니다.
