# 02 Role Assignment

팀 프로젝트는 역할을 나누되, 모든 팀원이 전체 흐름을 이해해야 합니다.

## 추천 역할

### Project Lead

- 전체 일정 관리
- 기능 범위 조정
- 발표 흐름 정리
- 팀원 진행 상황 확인

### Backend Developer

- FastAPI 라우터 구현
- Supabase 연결 코드 작성
- Pydantic 모델 작성
- API 테스트

### Frontend Developer

- Streamlit 화면 구현
- API 호출 코드 작성
- 상태 관리
- 사용자 입력 검증

### Supabase and Data Designer

- Supabase 테이블 설계
- SQL Editor 실행
- Auth/RLS 정책 확인
- 사용자별 데이터 접근 테스트

### Documentation and QA

- README 작성
- API 명세 작성
- 테스트 체크리스트 작성
- 오류 해결 기록 정리
- 발표 자료 정리

## 역할 운영 원칙

- 한 명이 모든 코드를 작성하지 않습니다.
- 각자 맡은 기능을 설명할 수 있어야 합니다.
- 매일 최소 2번 진행 상황을 공유합니다.
- 문제가 생기면 오류 메시지와 실행 명령을 함께 공유합니다.
- 발표자는 한 명이어도 되지만, 질문 답변은 팀 전체가 준비합니다.
