# 01 Topic Selection

팀별 Auto Healing 프로젝트 주제를 정합니다.

이번 과정의 기준 주제는 **에러 자가 치유(Auto Healing) 워크플로우**입니다. 팀별로 완전히 다른 서비스를 만들기보다, 같은 장애 복구 구조 안에서 장애 유형, Agent 역할, 복구 전략, 파이프라인 범위를 다르게 설계하는 것을 권장합니다.

## 기준 주제

```text
멀티 에이전트 협업 및 서비스 운영
-> 에러 자가 치유(Auto Healing) 워크플로우
```

## 작성 항목

- 프로젝트 이름
- 대상 서비스
- 비즈니스 요구사항
- 장애 시나리오 3개 이상
- Agent 역할 목록
- Agent 간 Handoff Context
- Docker Compose 서비스 목록
- 자동 복구 전략
- 파이프라인 단계
- 운영 대시보드에 보여줄 정보
- 알림과 에스컬레이션 기준

## 추천 변형 주제

- API 서버 장애 자동 복구 워크플로우
- LLM API Rate Limit 감지 및 대체 모델 호출 워크플로우
- DB 연결 고갈 감지 및 재연결 워크플로우
- Prompt Injection 감지 및 응답 차단 워크플로우
- Docker Compose 서비스 Health Check 및 재시작 워크플로우

## 좋은 주제 조건

- 장애 유형이 명확하다.
- 복구 전략이 2개 이상 있다.
- Agent 역할이 4개 이상으로 나뉜다.
- Docker Compose로 실행 가능한 서비스 구조가 있다.
- Health Check 또는 로그로 결과를 검증할 수 있다.
- 파이프라인 성공/실패 기준을 설명할 수 있다.
