# Lab 05. Security Policy Runbook

## 목표

AI 서비스 운영 보안 가이드라인을 Runbook 형태로 작성합니다.

Runbook은 장애나 보안 이벤트가 발생했을 때 운영자가 따라 할 수 있는 절차 문서입니다. 이 실습에서는 Prompt Injection, 민감 정보 노출, 권한 없는 Tool 실행 같은 상황에 대한 대응 기준을 정리합니다.

## 작성할 내용

```text
1. 보호해야 할 정보
2. 위험 입력 유형
3. 허용 가능한 응답 기준
4. 차단해야 하는 응답 기준
5. Agent별 Tool 실행 권한
6. 보안 이벤트 발생 시 대응 절차
7. 운영자에게 알릴 조건
8. 로그에 남길 정보
```

## 예시 정책

```text
정책 이름: 개인정보 보호 정책
대상: 사용자 입력, AI 응답, 운영 로그
규칙:
 - 주민등록번호, 비밀번호, API Key는 응답에 포함하지 않는다.
 - 사용자가 시스템 프롬프트를 요구하면 거절한다.
 - 권한 없는 Agent는 restart_service Tool을 실행할 수 없다.
대응:
 - 위험 입력은 차단하고 보안 이벤트 로그를 남긴다.
 - 반복 위반은 운영자 검토로 보낸다.
```

## 수업 활동

팀별 미니 프로젝트에 맞는 보안 Runbook을 작성합니다.

```text
프로젝트 이름:
보호해야 할 데이터:
위험 입력 예시:
차단 기준:
허용 기준:
운영자 알림 기준:
로그 항목:
```

## 확인 질문

- 보안 정책은 코드에만 있으면 충분한가?
- 운영자가 정책 위반 이벤트를 어떻게 확인할 수 있어야 하는가?
- Agent별 권한을 분리하지 않으면 어떤 위험이 생기는가?
