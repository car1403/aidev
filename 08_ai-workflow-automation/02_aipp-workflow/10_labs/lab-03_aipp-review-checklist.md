# Lab 03. AIPP 워크플로우 검토 체크리스트

## 목표

설계한 워크플로우가 실제 구현 가능한지 검토합니다.

## 체크리스트

```text
[ ] Trigger가 명확한가?
[ ] 입력 데이터가 충분한가?
[ ] LLM이 맡을 일과 Tool이 맡을 일이 분리되어 있는가?
[ ] Condition 분기 기준이 명확한가?
[ ] 실패했을 때 대체 경로가 있는가?
[ ] 실행 로그에 남길 값이 정의되어 있는가?
[ ] 개인정보나 민감정보 처리 기준이 있는가?
[ ] 결과를 사람이 검토해야 하는 조건이 있는가?
```

## 추가 질문

1. 이 워크플로우를 n8n으로 옮기면 어떤 노드가 필요할까요?
2. 이 워크플로우를 Dify로 옮기면 어떤 LLM 노드가 필요할까요?
3. AIPP에서 사람이 검토해야 하는 지점은 어디일까요?
