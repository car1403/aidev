# 02. AI Workflow Key Concepts

## AI 워크플로우란?

AI 워크플로우는 AI 모델 호출을 업무 흐름 안에 넣어 자동화하는 구조입니다.

```text
Trigger
-> Input
-> Condition
-> LLM
-> Tool
-> Memory
-> Action
-> Output
-> Log
```

## 핵심 용어

### Trigger

워크플로우를 시작하는 사건입니다.

예시:

```text
새 문의 접수
Webhook 호출
정해진 시간 도달
파일 업로드
버튼 클릭
```

### Condition

다음 경로를 결정하는 조건입니다.

예시:

```text
긴급도 high인가?
고객 등급이 premium인가?
답변 검증을 통과했는가?
문서 검색 결과가 있는가?
```

### Action

실제로 수행하는 작업입니다.

예시:

```text
메일 전송
Slack 알림
DB 저장
API 호출
담당자에게 전달
```

### LLM

텍스트를 이해하고 생성하는 AI 모델입니다.

적합한 작업:

```text
분류
요약
답변 초안 생성
문장 변환
의도 파악
```

### Tool

LLM이 직접 할 수 없는 실제 작업을 수행하는 기능입니다.

예시:

```text
주문 조회 API
문서 검색
계산 함수
데이터베이스 조회
알림 전송
```

### Memory

이전 대화, 사용자 정보, 실행 이력을 저장하는 구조입니다.

예시:

```text
사용자 선호 언어
이전 문의 이력
최근 실행 결과
대화 맥락
```

## 좋은 워크플로우의 기준

```text
각 단계의 역할이 명확하다.
입력과 출력이 구체적이다.
AI가 필요한 단계와 규칙 기반 단계가 구분되어 있다.
오류가 났을 때 대체 경로가 있다.
실행 로그가 남는다.
```
