# Test Checklist

팀 프로젝트 테스트 체크리스트입니다.

## 1. 기본 실행

```text
[ ] backend health check 통과
[ ] frontend 화면 접속 가능
[ ] 정상 입력 테스트
[ ] 긴급 입력 테스트
[ ] 잘못된 입력 테스트
```

## 2. 워크플로우 구조

```text
[ ] Trigger가 명확하다.
[ ] Input 항목이 정의되어 있다.
[ ] Condition 분기 기준이 있다.
[ ] AI 또는 LLM 역할이 정의되어 있다.
[ ] Tool/API/Knowledge 역할이 정의되어 있다.
[ ] Output 형식이 정리되어 있다.
```

## 3. 도구별 확인

```text
[ ] AIPP를 사용하는 경우 노드별 역할이 설명되어 있다.
[ ] n8n을 사용하는 경우 Webhook, IF, HTTP Request 흐름이 설명되어 있다.
[ ] Dify를 사용하는 경우 앱 유형, Prompt, Knowledge 연결이 설명되어 있다.
[ ] 선택한 도구의 실행 결과 또는 설계 캡처가 준비되어 있다.
```

## 4. 오류와 운영

```text
[ ] fallback 흐름 테스트
[ ] API 실패 시 처리 방식 확인
[ ] 잘못된 입력 처리 확인
[ ] 로그 기록 확인
[ ] 비용 또는 사용량 관리 계획 확인
[ ] 보안 주의사항 확인
```

## 5. 최종 시연

```text
[ ] 발표용 테스트 입력 준비
[ ] 기대 출력 준비
[ ] 실제 실행 결과 확인
[ ] 최종 시연 흐름 확인
[ ] 개선 방향 정리
```
