# Assignment 02 - 프롬프트 개선

이 과제에서는 모호한 개발 요청을 명확한 개발 요청으로 바꾸는 연습을 합니다.

Codex에게 좋은 답변을 얻으려면 기능 설명, 입력, 출력, 제약 조건, 완료 기준을 함께 전달해야 합니다. 단순히 "코드 만들어줘"라고 요청하면 결과가 원하는 방향과 달라질 수 있습니다.

## 목표

- 모호한 개발 요청의 문제점을 찾을 수 있다.
- 입력과 출력 조건을 명확히 작성할 수 있다.
- 예외 상황과 완료 기준을 요청문에 포함할 수 있다.
- Codex가 바로 작업할 수 있는 요청문을 작성할 수 있다.

## 개선할 원본 요청

```text
점수 계산하는 코드 만들어줘.
```

## 개선된 요청에 포함할 내용

- 만들 기능의 목적
- 함수명 또는 파일명
- 입력값 형식
- 출력값 형식
- 예외 상황
- 테스트 예시
- 완료 기준
- 초보자가 이해할 수 있는 주석 요청 여부

## 개선 요청문 예시

```text
Python 함수 하나를 작성해주세요.

기능:
여러 과목 점수를 입력받아 평균과 합격 여부를 계산합니다.

함수명:
calculate_score_result

입력:
- scores: 정수 점수 리스트

출력:
- 평균 점수
- 합격 여부

조건:
1. 빈 리스트가 들어오면 평균은 0, 합격 여부는 False로 반환합니다.
2. 평균이 60점 이상이면 합격입니다.
3. 점수는 0 이상 100 이하라고 가정합니다.
4. 초보자가 이해할 수 있도록 핵심 줄에 주석을 달아주세요.

테스트:
1. [80, 90, 100]
2. [40, 50, 60]
3. []
```
