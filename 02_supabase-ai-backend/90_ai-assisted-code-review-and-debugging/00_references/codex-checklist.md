# Codex 사용 체크리스트

## 요청 전

- [ ] 내가 만들 기능이나 해결할 문제를 한 문장으로 설명했다.
- [ ] 현재 파일 위치와 관련 파일을 정리했다.
- [ ] 입력과 출력을 정리했다.
- [ ] 제약 조건을 적었다.
- [ ] 완료 기준을 적었다.
- [ ] 바로 수정이 필요한지, 먼저 검토만 필요한지 정했다.

## 응답 확인

- [ ] 코드가 요구사항을 만족하는지 읽어봤다.
- [ ] 모르는 문법을 질문했다.
- [ ] 파일명과 실행 위치를 확인했다.
- [ ] 직접 실행해봤다.
- [ ] Codex 답변이 현재 폴더 구조와 맞는지 확인했다.

## 오류 발생 시

- [ ] 실행 명령을 기록했다.
- [ ] 오류 메시지를 그대로 복사했다.
- [ ] 기대 결과와 실제 결과를 구분했다.
- [ ] `.env` 위치와 변수명을 확인했다.
- [ ] 수정 후 같은 명령으로 다시 실행했다.

## 제출 전

- [ ] 내가 이해한 내용을 README 또는 보고서에 적었다.
- [ ] AI가 만든 코드 중 이해하지 못한 부분이 없다.
- [ ] 민감 정보가 코드와 문서에 들어 있지 않다.
- [ ] 최종 실행 결과를 확인했다.

## 백엔드 실습 코드 제출 전

- [ ] `.env` 파일을 GitHub에 올리지 않았다.
- [ ] `SUPABASE_SERVICE_ROLE_KEY`를 프론트엔드 코드에 넣지 않았다.
- [ ] `GEMINI_API_KEY`, `OPENAI_API_KEY`, `UPSTASH_REDIS_REST_TOKEN`을 출력하지 않았다.
- [ ] mock-first, Gemini SDK 최소/안내형 예제, OpenAI 선택 사용 흐름을 구분했다.
- [ ] 실제 LLM API 호출 예제와 mock-first 예제를 구분했다.
- [ ] update/delete 코드에 `.eq(...)` 같은 조건이 들어 있다.
- [ ] FastAPI endpoint의 오류 응답을 확인했다.
- [ ] Supabase 저장 결과를 Table Editor 또는 API 응답으로 확인했다.
- [ ] Redis key에 필요한 TTL이 설정되어 있다.
- [ ] 서비스 로그에 민감 정보가 들어가지 않는다.
