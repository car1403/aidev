# Codex 요청 패턴

Codex에게 요청할 때는 원하는 결과만 말하기보다 현재 상태, 제약 조건, 완료 기준을 함께 전달하는 것이 좋습니다.

## 기본 패턴

```text
무엇을 하려는지:

현재 상태:

관련 파일:

원하는 결과:

제약 조건:

완료 기준:

아직 코드를 수정하지 말고 먼저 검토만 할지, 바로 수정할지:
```

## 코드 생성 요청

```text
Python으로 점수 리스트의 평균을 계산하는 함수를 만들어주세요.

조건:
1. 함수 이름은 calculate_average입니다.
2. 숫자 리스트를 인자로 받습니다.
3. 평균을 반환합니다.
4. 빈 리스트가 들어오면 0을 반환합니다.
5. 초보자가 이해할 수 있도록 핵심 줄에 주석을 달아주세요.

테스트 예시도 3개 함께 작성해주세요.
```

## 코드 설명 요청

```text
아래 코드를 초보자 기준으로 설명해주세요.

요청:
1. 전체 코드의 목적을 먼저 설명해주세요.
2. 각 줄이 무슨 역할을 하는지 알려주세요.
3. 변수에 어떤 값이 들어가는지 예시를 들어주세요.
4. 마지막에 꼭 기억해야 할 개념을 정리해주세요.

아직 코드를 수정하지 말고 설명만 해주세요.
```

## 디버깅 요청

```text
아래 명령을 실행했더니 오류가 발생했습니다.

실행 명령:
python starter.py

오류 메시지:
여기에 오류 메시지를 붙여 넣습니다.

내가 기대한 결과:

요청:
1. 오류 원인을 설명해주세요.
2. 가장 먼저 확인할 것을 알려주세요.
3. 수정 방향을 제안해주세요.
4. 수정 후 다시 확인할 명령을 알려주세요.
```

## 리팩토링 요청

```text
아래 코드를 리팩토링해주세요.

조건:
1. 동작은 바꾸지 말아 주세요.
2. 함수로 나눌 수 있는 부분을 분리해주세요.
3. 변수명과 함수명을 더 명확하게 바꿔주세요.
4. 너무 어려운 고급 문법은 피해주세요.
5. 변경 전/후 확인 방법을 알려주세요.
```

## 코드 리뷰 요청

```text
아래 코드를 리뷰해주세요.

리뷰 우선순위:
1. 실행 오류 가능성
2. 요구사항 누락
3. 보안 위험
4. 예외 처리 부족
5. 중복 코드와 가독성

출력 형식:
- 치명적인 문제
- 수정이 필요한 문제
- 개선하면 좋은 문제
- 확인 질문
- 실행 또는 테스트 방법

아직 코드를 수정하지 말고 리뷰 결과만 정리해주세요.
```

## 백엔드 리뷰 요청

```text
아래 FastAPI/Supabase/Redis 코드를 리뷰해주세요.

리뷰 관점:
1. endpoint URL과 HTTP Method가 적절한가?
2. Pydantic 요청/응답 모델이 충분한가?
3. Supabase 테이블명과 컬럼명이 일치하는가?
4. update/delete 조건 누락 위험이 있는가?
5. Upstash Redis key와 TTL 설계가 적절한가?
6. API key, service role key, Redis token 노출 위험이 있는가?
7. 서비스 로그에 민감 정보가 들어갈 가능성이 있는가?
```
