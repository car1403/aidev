# Lab 04 - 스크립트 리팩토링

이 lab에서는 긴 Python 스크립트를 함수 중심 구조로 리팩토링하는 방법을 연습합니다.

리팩토링의 핵심은 동작을 바꾸지 않고 구조를 개선하는 것입니다. 먼저 현재 코드가 어떤 결과를 내는지 확인한 뒤, 함수로 나누고 다시 같은 결과가 나오는지 비교합니다.

## 실습 목표

- 리팩토링 전 실행 결과를 기록할 수 있다.
- Codex에게 동작 유지 조건을 명확히 전달할 수 있다.
- 긴 코드를 작은 함수로 나눌 수 있다.
- 리팩토링 후 결과가 달라지지 않았는지 확인할 수 있다.

## 실습 파일

```text
before.py
```

## 실습 절차

1. `before.py`를 먼저 실행합니다.
2. 실행 결과를 기록합니다.
3. 코드에서 반복되는 부분과 이름이 필요한 부분을 표시합니다.
4. Codex에게 리팩토링을 요청합니다.
5. 생성된 코드가 기존 결과와 같은지 확인합니다.
6. 함수명이 역할을 잘 설명하는지 확인합니다.
7. 너무 복잡한 구조로 바뀌지 않았는지 검토합니다.

## Codex 요청 예시

```text
아래 Python 스크립트를 리팩토링해주세요.

조건:
1. 실행 결과는 기존 코드와 같아야 합니다.
2. 코드를 함수 중심으로 나누어 주세요.
3. 함수명은 역할이 드러나도록 작성해주세요.
4. 초보자가 이해하기 어려운 고급 문법은 피해주세요.
5. 수정 후 어떤 함수를 어떤 순서로 호출하는지 설명해주세요.

먼저 리팩토링 방향을 설명한 뒤 코드를 제안해주세요.
```

## 검토 기준

- 기존 실행 결과와 리팩토링 후 실행 결과가 같은가?
- 함수가 한 가지 역할만 담당하는가?
- 함수명이 역할을 설명하는가?
- 불필요하게 복잡한 클래스나 고급 문법이 추가되지 않았는가?
- 주석이 핵심 흐름을 이해하는 데 도움이 되는가?

## 결과 정리

```md
## Lab 04 결과

- 동작이 유지되었는지:
- 구조가 더 읽기 쉬워진 이유:
```
