# Lab 07 - 환경 변수와 API Key 오류 디버깅

이 lab에서는 `.env` 설정 누락, 변수명 오타, API key 또는 token 출력 위험을 Codex와 함께 점검합니다.

환경 변수 오류는 코드 문법 오류와 다릅니다. 파일 위치, 변수명, 실제 값, 실행 위치가 모두 맞아야 정상 동작합니다.

## 실습 목표

- `.env`와 `.env.example`의 차이를 설명할 수 있다.
- 환경 변수 이름 불일치 문제를 찾을 수 있다.
- API key와 token을 출력하면 안 되는 이유를 설명할 수 있다.
- Codex에게 실제 key를 붙여 넣지 않고도 오류 분석을 요청할 수 있다.

## 실습 파일

```text
broken_env_loader.py
```

## 실습 절차

1. `broken_env_loader.py`를 읽습니다.
2. 코드가 어떤 환경 변수를 읽는지 표시합니다.
3. `.env.example`에 어떤 변수명이 필요할지 정리합니다.
4. key를 그대로 출력하는 코드가 왜 위험한지 설명합니다.
5. Codex에게 디버깅 요청 템플릿으로 질문합니다.
6. 수정 방향을 정리합니다.
7. 실제 key는 문서나 채팅에 붙여 넣지 않습니다.

## Codex 요청 예시

```text
아래 환경 변수 로딩 코드에 문제가 있습니다.

상황:
- Supabase URL이 없다고 나옵니다.
- API key가 그대로 출력될 위험이 있습니다.
- 실제 key 값은 붙여 넣지 않겠습니다.

확인 요청:
1. 코드에서 읽는 환경 변수 이름을 정리해주세요.
2. .env.example에 어떤 항목이 있어야 하는지 알려주세요.
3. 실제 key를 print하면 왜 위험한지 설명해주세요.
4. 안전하게 확인하는 방법을 알려주세요.
5. 초보자가 따라 할 수 있는 수정 순서를 알려주세요.

아직 코드를 수정하지 말고 원인과 확인 순서를 먼저 정리해주세요.
```

## 안전 기준

- 실제 API key를 Codex 요청문에 붙여 넣지 않습니다.
- `.env` 파일은 GitHub에 올리지 않습니다.
- `.env.example`에는 실제 key 대신 예시 값을 넣습니다.
- key가 필요한지 확인할 때는 값 전체를 출력하지 않습니다.
- `있음/없음` 정도만 확인하는 방식이 안전합니다.
