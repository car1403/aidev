# Lab 03 - 오류 메시지 디버깅

이 lab에서는 Python 오류 메시지를 Codex에게 전달하고, 원인과 수정 방법을 단계적으로 확인합니다.

오류가 발생했을 때 가장 먼저 할 일은 코드를 무작정 바꾸는 것이 아니라, 실행한 명령과 오류 메시지를 정확히 남기는 것입니다.

## 실습 목표

- 오류 메시지를 끝까지 읽는 습관을 만든다.
- 실행 명령, 파일 위치, 오류 메시지를 함께 정리한다.
- Codex에게 원인 후보와 확인 순서를 요청할 수 있다.
- 수정 후 같은 명령으로 다시 실행해 결과를 확인한다.

## 실습 파일

```text
broken.py
```

## 실습 절차

1. `broken.py`를 실행합니다.
2. 터미널에 출력된 오류 메시지를 복사합니다.
3. 오류가 발생한 파일명과 줄 번호를 확인합니다.
4. Codex에게 디버깅 요청 템플릿으로 질문합니다.
5. Codex 답변에서 원인 후보와 수정 방법을 구분합니다.
6. 코드를 수정합니다.
7. 같은 명령으로 다시 실행합니다.
8. 오류 원인을 짧게 기록합니다.

## Codex 요청 예시

```text
Python 파일을 실행했더니 오류가 발생했습니다.

실행한 위치:
C:\aidev\02_supabase-ai-backend\90_ai-assisted-code-review-and-debugging\10_labs\lab03_debug-error-message

실행한 명령:
python broken.py

오류 메시지:
여기에 터미널 오류 메시지를 그대로 붙여 넣습니다.

요청:
1. 오류 메시지를 초보자 기준으로 설명해주세요.
2. 어떤 줄에서 문제가 발생했는지 알려주세요.
3. 원인 후보를 2~3개로 나누어 주세요.
4. 가장 먼저 확인할 것을 알려주세요.
5. 수정 예시를 보여주세요.
6. 수정 후 어떻게 다시 확인하면 되는지 알려주세요.
```

- [ ] Codex에게 바로 수정이 아니라 원인 설명을 먼저 요청했다.
