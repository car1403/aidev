# Lab 05 - 내 코드 리뷰 요청

이 lab에서는 직접 작성한 Python 코드를 Codex에게 리뷰하도록 요청하고, 리뷰 결과를 우선순위별로 분류합니다.

코드 리뷰는 모든 제안을 무조건 반영하는 작업이 아닙니다. 실행 오류나 요구사항 누락처럼 중요한 문제를 먼저 확인하고, 변수명이나 중복 코드 개선은 그 다음에 봅니다.

## 실습 목표

- 직접 작성한 코드를 리뷰 대상으로 준비할 수 있다.
- Codex에게 리뷰 관점을 구체적으로 지정할 수 있다.
- 리뷰 결과를 치명도별로 분류할 수 있다.
- 반영할 수정과 보류할 수정을 구분할 수 있다.

## 준비할 코드

다음 중 하나를 선택합니다.

- Python 기초 단원에서 작성한 계산기 예제
- 파일 입출력 예제
- 함수 실습 예제
- FastAPI 이전 단계의 간단한 Python 스크립트

처음에는 너무 긴 코드보다 30~80줄 정도의 코드를 선택하는 것이 좋습니다.

## Codex 요청 예시

```text
아래 Python 코드를 리뷰해주세요.

코드 목적:
사용자가 입력한 값을 처리하고 결과를 출력하는 연습용 코드입니다.

리뷰 관점:
1. 실행 오류 가능성
2. 요구사항 누락
3. 입력값 검증 부족
4. 변수명과 함수명
5. 중복 코드
6. 초보자가 이해하기 어려운 부분

출력 형식:
- 반드시 수정할 문제
- 수정하면 좋은 문제
- 지금은 보류해도 되는 문제
- 확인 질문

아직 코드를 수정하지 말고 리뷰 결과만 정리해주세요.
```

## 리뷰 결과 분류 기준

| 구분 | 의미 |
| --- | --- |
| 반드시 수정 | 실행 오류, 잘못된 결과, 요구사항 누락 |
| 수정하면 좋음 | 중복 코드, 함수명, 변수명, 구조 개선 |
| 보류 가능 | 현재 단계에서 너무 어려운 고급 구조 |
| 확인 질문 | 코드만 보고 판단하기 어려운 요구사항 |

## 결과 정리

```md
## Lab 05 결과

- 리뷰한 파일:

- 수정하면 좋은 문제:
- 보류한 문제:
- 직접 반영한 수정:
- 수정 후 실행 결과:
```

## 완료 체크리스트

- [ ] 리뷰 관점을 구체적으로 지정했다.
- [ ] 리뷰 결과를 우선순위별로 분류했다.
- [ ] 반영할 수정과 보류할 수정을 구분했다.
